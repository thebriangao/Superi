# P3.W06.C002

## Outcome

Implemented native desktop menus, project dialogs, project and media drag and drop, platform
clipboard roles, recent projects, document titles, workspace restoration, and a safe close path.
The shell preserves active document identity and recent paths across launches, saves before project
replacement or application exit, and keeps session-history loss visible before the current undo
stack ends.

## Architecture and integration

One native `DesktopShellState` accepts a bounded, sequence-fenced presentation from the primary
React application owner. It builds the process-wide File, Edit, and Workspace menus, maps stable
menu identities to typed intents delivered only to the focused editor webview, projects only a path
basename and durable revision into the primary native title, persists registry-reconciled panel
presentation privately, and admits exactly one close resolution into the existing orderly
application lifecycle. The window-session owner retains each webview's active route and placement.
Primary window close, menu quit, and operating-system process quit all enter the same handshake.
Native cut, copy, paste, and select-all roles retain platform text behavior.

The existing `DesktopProjectState` remains the sole desktop document owner. It now atomically
persists a private versioned session record containing the active path and bounded last-known recent
records, validates the active project through `LocalProjectHost` on launch, and degrades to no active
document with actionable failure state when that path cannot be reopened. Project bytes, document
identity, settings, recovery, and revision changes remain below this shell boundary.

An always-mounted application consumer reconciles persisted panel identities against the live
registry without replacing the window session's route, synchronizes project identity and
engine-owned undo or redo depth into native menu state,
uses native open and save dialogs, separates one `.superi` document drop from media-only drops, and
routes history commands through the existing generated project command. Before closing a document
with live session history it explains that the durable project will be saved while the in-memory
undo and redo history ends; active work blocks closing, and approval is single-use.

## Implementation process

Failure-first Rust contracts established native synchronization, recent-menu mapping, workspace
persistence, stale sequence handling, invalid-state rejection, one-shot close resolution, active
project restoration, missing-document degradation, and nonblocking presentation-record failure
before production behavior existed. Pure
TypeScript contracts established drag and drop partitioning, safe-close decisions, path-safe native
titles, webview sequence resumption, and live-registry workspace reconciliation before the frontend
adapter and consumer were implemented.

project, public API, engine history, application registry, media import, Tauri lifecycle, and GPU
viewport owners were retained rather than duplicated.
