# P1.W07.C018

## Outcome

Created a tiny deterministic raw-video baseline spanning every supported pixel format and standard
frame rate. The versioned baseline contains 207 one-frame cases at 5 by 3 pixels, exercises odd
chroma geometry, and remains below 64 KiB.

## Architecture and integration

`superi-fixture-tool` owns a dependency-free generator and `generate-video` command that creates a
new fixture directory without overwriting existing evidence. It emits a fixed CSV catalog, one
contiguous binary payload, and a schema 1 manifest with exact provenance, sizes, and hashes. The
`superi-media-io` contract derives the expected matrix from `PixelFormat::ALL` and the standard
`FrameRate` constants, validates every catalog record and sample representation, and constructs
each case through `VideoPlane`, `CpuVideoBuffer`, `VideoFormat`, and `VideoFrame`.

## Implementation process

The verified base was `c6c7066`. Generator and consumer contracts were written first. The generator
contract failed because no generator API existed, while the media consumer compiled and failed
because the canonical video catalog was absent. The implementation then added explicit raw-plane
geometry, deterministic finite sample synthesis, stable catalog serialization, no-overwrite CLI
behavior, canonical artifacts, and real consumer proof.

## Verification

Focused generator and CLI contracts prove byte-for-byte reproduction, the 207-case report,
sub-64-KiB payload size, usage diagnostics, and preservation of existing output. The media-I/O
contract proves the complete 23 by 9 matrix, exact rational rates, contiguous offsets, per-plane
hashes, packed and subsampled geometry, finite floating-point data, 10-bit bounds, P010 alignment,
and public frame construction. The canonical fixture validator covers manifest policy, provenance,
inventory, byte counts, and payload digests.

## Delivery

This record is committed with the generator, CLI, canonical artifacts, consumer contracts,
fixture documentation, and current codebase maps. The delivery commit containing this record is
pushed to `origin/main`; its SHA is recorded in the matching Google Docs completion description and
ignored execution log after it exists.

## Remaining constraints

These raw single-frame fixtures do not claim encoded-codec round trips, CFR or VFR sequences, HDR
reference media, malformed inputs, hardware behavior, audio synchronization, or the complete
editorial slice. Those remain separate checkpoints and can consume this stable baseline where the
format and timing matrix is relevant.
