# P3.W07.C004

## Outcome

Implemented one immutable application semantic contract shared by macOS, Windows, and Linux across
the exact project, engine, UI, shortcut, media, color, audio, AI, plugin, and export domains. The
System route exposes this parity boundary while keeping platform availability and physical adapter
behavior honest and separate.

## Architecture and integration

`app/src/platform-parity.ts` owns the schema-1 `superi.desktop.semantic-parity.v1` contract. Every
supported platform projects the same deeply frozen ordered domain objects, so no target can rename
commands, omit a semantic area, or reinterpret state while still claiming the shared contract.

`PlatformParityPanel` is a read-only System-route consumer. It names all supported platforms,
reports the common contract identity, and exposes each domain invariant without detecting hardware,
branching on the current operating system, or taking authority from the existing project, engine,
workspace, color, audio, AI, plugin, or delivery owners.

## Implementation process

Focused contracts first failed because the parity owner and System consumer did not exist.
Implementation added the exact platform and domain catalogs, runtime rejection for unsupported
labels, stable display names, immutable projections, and the visible production consumer before
widening into the existing frontend gates.

## Verification

Focused parity and application source contracts cover all three platforms, all ten domains, unique
coverage, identical object identity, deep freezing, unsupported-target rejection, the audio timing
and routing invariant, production registration, and native-import exclusion. The complete frontend
suite passes 181 tests, strict TypeScript passes, the production build transforms 100 modules, and
all 26 module maps plus the global index validate with clean diff checks.

## Delivery

The shared parity owner, System consumer, focused tests, package gate, checkpoint record, and
refreshed maps are delivered together by the commit containing this file. The matching live
checklist records the exact commit only after origin/main containment is confirmed.

## Remaining constraints

Capability availability may differ by machine and adapter without changing application semantics.
Platform-specific GPU, audio, filesystem, font, monitor, and codec adapters remain separate from
this contract, and physical macOS, Windows, and Linux certification remains hosted or device-lane
evidence rather than a claim made by the static catalog.
