# P3.W01.C005

## Outcome

Implemented the desktop application framework with deterministic routing, transient workspace
layout, explicit panel and command registries, and one immutable shared public-resource selection.

## Architecture and integration

`ApplicationRegistry` validates unique routes, panels, commands, shortcuts, and cross-references at
construction. Its reducer owns only transient route, visibility, focus, and typed selection state;
it does not own authoritative project data, persistence, media, audio, engine, or transport behavior.

`ApplicationProvider` composes that model above C003's injected `SuperiApiBindings` and C004's
unchanged reliable transport. The minimal React shell consumes registry definitions for generic
workspace and system routes, exposes one selection across panels, and delegates asynchronous engine
commands through the generated client while applying local state changes before awaiting transport.

## Implementation process

Implementation began from synchronized dependency commit
`7be93cb968ee31f925ece62729c792cc66535a6d`, which contains C001 through C004. The focused framework
contract first failed because `app/src/application.ts` did not exist. The implementation then added
the pure application model, its React provider, one minimal shell consumer, responsive presentation,
and focused composition contracts without changing the delivered transport or native bridge.

## Verification

The focused application framework contract passes five cases covering registry validation, route
and panel reconciliation, immutable typed selection, local-first asynchronous generated API command
delegation, shortcut normalization, and editable-target safety. The complete frontend suite passes
17 cases, strict TypeScript checking passes, and the locked production Vite bundle builds. The
codebase map validator passes after the source-backed map refresh.

## Delivery

The application model, React provider and shell consumer, contracts, maps, and this record are
delivered together by the commit containing this file. The exact delivered commit is recorded in
the matching checklist completion description after `origin/main` containment is confirmed.

## Remaining constraints

Workspace layout and selection are intentionally transient and nonauthoritative. Professional
editing and compositing workspaces, project persistence, broad generated method routing, and
engine-owned audio timing, channel meaning, synchronization, routing intent, and audible continuity
remain outside this checkpoint and are neither duplicated nor mocked by the application framework.
