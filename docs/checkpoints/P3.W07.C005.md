# P3.W07.C005

## Outcome

Implemented a strict current-host declaration for GPU, audio, filesystem, font, monitor, and codec
adapter families behind six stable shared contracts. macOS, Windows, and Linux retain distinct
native implementations while media timing, precision, metadata, alpha, and predictable fallback
remain one explicit cross-platform guarantee set.

## Architecture and integration

`app/src-tauri/src/platform_adapters.rs` owns a dependency-free schema-1 snapshot. A compile-target
selector chooses one of three fully tested adapter sets, and the read-only
`desktop_platform_adapters` command returns exact implementation and contract identities without
probing hardware, opening devices, mutating files, loading fonts, moving monitors, or starting codec
sessions.

`app/src/platform-adapters.ts` strictly parses the native snapshot. It rejects unknown fields,
unsupported targets, incomplete or duplicate domain order, changed contract identities, malformed
implementation names, and guarantee drift before deeply freezing accepted state.
`PlatformAdaptersPanel` exposes the current declaration in System beside, but separate from, live
hardware capability discovery.

## Implementation process

Focused frontend contracts first failed because neither native nor TypeScript adapter owners
existed. Implementation added all-target Rust fixtures and the current-target command, registered it
with the existing Tauri builder, added the strict frontend boundary, and mounted the read-only
consumer. The first native proof populated the previously empty locked Tauri build cache; no broad
native suite was repeated after the two focused tests passed.

## Verification

Focused Node contracts cover one native invocation, exact six-domain order, shared contract IDs,
deep immutability, target rejection, duplicate and contract drift, unknown fields, native command
registration, and production consumption. Two focused Rust tests cover every platform declaration,
identical guarantee and contract order, distinct implementations, and compile-target selection.
Rust formatting and strict TypeScript pass, the complete reconciled frontend suite passes 193
tests, the production build transforms 104 modules, and all 26 module maps plus the global index
validate with clean diff checks.

## Delivery

The native snapshot, strict frontend parser, System consumer, focused tests, package gate,
checkpoint record, and refreshed maps are delivered together by the commit containing this file.
The matching live checklist records the exact commit only after origin/main containment is
confirmed.

## Remaining constraints

Declarations identify implementation families, not physical availability. Existing capability and
runtime owners remain authoritative for devices, drivers, routes, filesystems, fonts, monitors, and
codec execution; physical platform certification remains hosted or device-lane evidence.
