# P1.W07.C006

## Outcome

Superi now blocks pushes and pull requests when the locked open workspace uses a dependency outside
the approved permissive license allowlist or trusted source policy. The CI job also runs a focused
repository contract check before invoking cargo-deny.

## Architecture and integration

The read-only GitHub Actions workflow audits `open/Cargo.toml` with all features so target-specific,
optional, build, and runtime dependency edges are visible. `open/deny.toml` remains the single policy
source, including the zero-copyleft allowlist, revision-only Git requirement, and explicit OxideAV
source approval. The executable checker connects workflow permissions, action inputs, and the
canonical deny policy without adding any runtime dependency or network path to the open product.

## Implementation process

The verified base was `a11cecd37bfa9338cf3de933e5552669f0df8df4`. Repository policy files,
Cargo manifests, build scripts, lock metadata, governing licensing documents, and representative
consumer modules were inspected before implementation. A missing-workflow assertion established the
original RED state, then the workflow and directly stale documentation were added. The durable
follow-up established a second RED state for the absent executable contract step before wiring the
checker into the real CI job.

## Verification

The focused shell contract, YAML parsing, all-feature cargo-deny license and source checks, Rust
formatting, diff validation, and prohibited prose-dash scan passed locally. The original delivered
workflow also completed successfully in GitHub Actions run `29302533491`. Cargo commands use
`CARGO_TARGET_DIR=/private/tmp/superi-shared-target` to avoid duplicated generated output.

## Remaining constraints

Cargo-deny relies on dependency metadata and license-file detection, so automated success does not
replace the required human and legal review. The configured `Unicode-DFS-2016` allowance is currently
unused and produces a non-failing warning. Advisory and duplicate-version policy checks remain
outside this checkpoint, which intentionally runs only the approved license and source checks.
