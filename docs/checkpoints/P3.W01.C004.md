# P3.W01.C004

## Outcome

Implemented the production desktop command and event transport with ordered replay, reconnection,
cooperative cancellation, and actionable classified public failures above the existing managed
engine connection.

## Architecture and integration

The native `DesktopTransportState` accepts connect, request, cancel, and disconnect variants through
one asynchronous `desktop_api_dispatch` Tauri command. It routes the existing generated integration
validation method into C002's unchanged `EngineConnection`, waits only on a blocking-safe worker,
retains a bounded ordered engine-introspection replacement stream, and converts errors through the
canonical `PublicApiError` surface with reviewed safe context. Unrouted generated methods return an
honest degraded failure instead of duplicating engine semantics.

`DesktopSuperiTransport` satisfies the generated `SuperiTransport` contract through Tauri invoke and
listen, preserves request identities, rejects stale and duplicate event delivery, reconnects with the
last accepted cursor, replays retained replacements once, cancels abandoned generation-scoped work,
and throws the generated `SuperiTransportError` without losing context or last-valid resource data.
C003's provider remains the only `SuperiClient` factory, while React consumes generated validation
and introspection state only for responsive presentation.

## Implementation process

Implementation began from synchronized dependency commit
`3b51b5db89ce5e74419366f741c0ca6f31a31a08`, which contains C001 through C003. Focused native and
frontend contracts first failed because the C004 transport modules did not exist. The implementation
then added the thin native dispatcher, concrete frontend state machine, fixed bootstrap injection,
real generated validation consumer, and focused proof while preserving C002's `engine.rs` and the
canonical generated TypeScript artifact byte-for-byte.

## Remaining constraints

The current C002 connection exposes only integration validation, so all other generated methods
remain structured degraded failures until their engine-owned routes exist. Cancellation is
cooperative: an already admitted validation query may finish internally, but its abandoned result is
discarded and never delivered. The transport carries only commands and compact generated state
events; it adds no media payload IPC, editor semantics, project ownership, or duplicate API client.
