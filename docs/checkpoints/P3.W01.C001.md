# P3.W01.C001

## Outcome

Added the production React and Tauri application shell with explicit, revisioned application and
headless-engine lifecycle state for startup, orderly shutdown, restart, recovery, and classified
failure.

## Architecture and integration

`ApplicationLifecycle` is the sole desktop lifecycle owner. It wraps the established
`superi-concurrency::LifecycleCoordinator`, records application intent and engine generation, and
exposes one exact `HeadlessEngineLifecycleParticipant` seam for the adjacent engine process owner.
Every engine transition requires the coordinator's current signal token, so stale completion cannot
advance a new lifecycle generation.

Tauri owns only shell-local lifecycle snapshot and request commands. The React client renders those
safe serializable snapshots asynchronously and presents retry, restart, and orderly quit actions
without importing generated engine bindings, starting a process, or duplicating engine behavior.
The native exit callback records shutdown intent without blocking Tauri's main thread, while a
blocking-safe monitor exits only after the headless-engine participant acknowledges the stop.
