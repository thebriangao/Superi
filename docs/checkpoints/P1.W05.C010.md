# P1.W05.C010

## Outcome

Implemented explicit luminance-preserving tone mapping for authoritative display and deliverable transforms, plus a separate legal-range RGB storage encoder for 8 through 16-bit signals.

## Architecture and integration

`OutputTransformOptions` now selects either no tone mapping or a validated rational luminance shoulder. `OutputColorTransform` applies that shoulder after destination-primary and gamut conversion and before transfer encoding, using one RGB scale factor to retain hue ratios. `LegalRangeEncoder` remains downstream of the full-range premultiplied output boundary: it unassociates RGB, rejects extended values instead of clipping, rounds to normative legal integer codes, preserves alpha and artifact identity, and emits normalized limited-range straight-alpha binary32 storage without choosing a YUV matrix.

## Implementation process

Work began from the C003 output-transform and C005 color-rule foundations, with failing public contracts added before implementation. A retired draft was preserved by checksum, reconciled through ordinary patches, and reworked against the authoritative APIs. The implementation was repeatedly synchronized without discarding incoming vertical-slice, workflow, fixture, scenario, expectation, instrumentation, lockfile, or map changes. The final verified base was `fcd7f599415b3c6a73f14d7b6912996c13dd71f8`.

## Verification

Focused contracts cover identity below the tone-map knee, exact peak mapping, RGB-ratio preservation, display and delivery rule consumption, exact 8, 10, and 12-bit anchors, deterministic midpoint rounding, alpha semantics, stage separation, invalid configuration, and extended-value rejection. Fresh proof also covers the complete color suite and deterministic color fixture, formatting, locked workspace builds and tests with the repository's documented macOS native-codec exclusions, strict Clippy, color rustdoc with warnings denied, Rust 1.80 compatibility, fixture validation, dependency direction, open-boundary policy, network-isolated workflow structure, and codebase-map validation.

## Delivery

The implementation, tests, map updates, and this record are delivered together in one lowercase three-sentence commit to `origin/main`. The exact delivered SHA is recorded by repository history and the matching Google Docs completion description after remote containment is confirmed.

## Remaining constraints

The encoder intentionally stops at normalized legal-range RGB. YUV matrix conversion, chroma subsampling, packed integer storage, GPU output transforms, executable ICC evaluation, project-persisted color configuration, and production engine export or viewport orchestration remain outside this checkpoint.
