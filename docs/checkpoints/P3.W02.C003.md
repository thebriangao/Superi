# P3.W02.C003

## Outcome

Implemented source, program, composite, and color viewer roles on the existing native GPU
presentation path. Each role owns a native child surface and a managed canonical GPU render result,
with role-aware geometry and status projected into the matching professional workspace.

## Architecture and integration

The one application-owned `GpuSubmission` thread still owns adapter, device, resource, submission,
surface, and teardown order. It now serializes four role-addressed native surfaces and presents each
managed `Rgba16Float` texture through C001's unchanged `GpuDisplayPresenter`; React sends only role,
geometry, scale, and visibility through C002's strict unknown-field-denying placement DTO.

`ViewerPresentationIntent` retains a scene-linear ACEScg source pipeline and derives one terminal
sRGB display branch without mutating the scene metadata. Its explicit contract fixes premultiplied
alpha, RGBA16F storage, alpha unassociation, primary conversion, gamut mapping, tone mapping,
transfer encoding, alpha reassociation, display identity, arbitrary supported extent, and
deterministic equality for repeated intent.

## Implementation process

Implementation began after C001 and C002 were remotely contained and synchronized at
`1940765a5d9b3e887b5dfd3791c20b303cebe4e6` and
`6ddba71dc0d9d62d062185731f5e38a9f1a8bf1e`. The frozen focused contract first failed because the
four role and presentation-intent types did not exist, then passed after the smallest native-host
extension. Editing now consumes source and program viewers, compositing consumes composite, and
color consumes color without adding media IPC, a webview image path, a second GPU owner, or a new
display transform implementation.

## Verification

The focused Rust role and color contract passes one case covering all four roles, exact scene and
display meaning, transform order, deterministic intent, canonical precision and alpha, 8K extent,
and invalid zero extent. The focused app checks pass four cases covering exact workspace placement,
the existing application owner, preserved audio projection, and C002's control-only native viewport
IPC boundary. Rust formatting passes, and the codebase map validator passes after the final
source-backed workspace map and global index update.

## Delivery

The native role registry, immutable color intent, four workspace consumers, focused contracts,
maps, and this record are delivered together by the commit containing this file. The exact delivered
commit is recorded in the matching checklist completion description after `origin/main` containment

## Remaining constraints

The current application has no upstream media-import or timeline render producer, so each role starts
with a distinct managed canonical GPU result owned by the native host. Later media and playback
checkpoints may replace those initial results on the same role, color, resource, and presentation
boundary without introducing frame IPC; monitor ICC and HDR policy remain owned by P3.W05.C009.
