# P1.W07.C003

## Outcome

Added explicit default and `os-codecs` consumer coverage to the cross-platform Rust workflow. The
supported macOS, Windows, and Ubuntu 26.04 matrix lanes now build the real CLI feature path and test
the engine and API consumers while every existing lane retains the complete default quality suite.

## Architecture and integration

The feature path remains `superi-cli` to `superi-api` to `superi-engine` and finally the optional
`superi-codecs-platform` adapter. Ubuntu 24.04 and the scheduled Ubuntu 22.04 job remain
default-only because their distribution libva API is below the platform crate's required version;
Windows coverage remains enabled and unfiltered.

The default codec graph now pins crates.io `rav1d` 1.1.0 with assembly disabled and only the 8-bit
and 16-bit decode features enabled. Its BSD-2-Clause license and Rust 1.79 floor satisfy repository
policy, and its packaged build script removes the broken rav1d 1.0.0 MSVC reference to an excluded
`tools/compat/getopt.c` source file. The decoder also performs a checked conversion between Superi's
64-bit sequence identity and rav1d's target-specific C offset width, which is 32-bit on Windows
MSVC, while widening decoded offsets losslessly for timing correlation. C002's libva, nasm, and
checksum-pinned libvpx 1.16.0 setup and C009's network-isolated workflow remain intact.

## Implementation process

A focused repository contract was added first and failed because the synchronized workflow had no
explicit feature policy. Upstream release metadata and immutable crates.io package contents were
reviewed before the central exact dependency and lockfile checksum changed. Focused AV1 and default
registry tests then passed before the workflow received its matrix properties and gated CLI,
engine, and API commands.

## Verification

The focused CI contract and YAML parsing passed. Exact rav1d archive checksum verification, Cargo
feature inspection, the dependency-policy workflow contract, cargo-deny license and source checks,
seven AV1 contracts, three registry contracts, default and `os-codecs` CLI builds, and the
`os-codecs` engine and API suites passed from the checkpoint target.

Restored hosted execution exposed and reproduced the rav1d C offset width mismatch on Windows
Server 2025 before the checked conversion was added. Fresh formatting plus all seven AV1 and three
default-registry contracts pass locally after the correction; the unchanged hosted Windows build
and complete descendant matrix remain the final delivery gate.

The locked workspace build, hosted-macOS-equivalent workspace test suite, strict all-target Clippy,
documentation tests, formatting, and the open-tree boundary scan also passed after integration with
the portable hosted-slice baseline. All ten fixture versions, 23 payloads, the canonical scenario,
and the media, color, image-sequence, timing, OTIO, and expectation consumers passed. The local
workspace suite retained only C002's three named physical VideoToolbox and AudioConverter
exclusions; their tests and owning hardware-lane requirement were not changed.
