# P2.W04.C002

## Outcome

Superi audio now discovers real operating-system output devices and starts low-latency typed output
streams through CoreAudio, WASAPI, or ALSA. The engine-facing sample path is explicitly bounded,
preserves complete interleaved frames, outputs timed silence on starvation, and publishes the exact
physical device-frame position through the shared audio master clock.

## Architecture and integration

`superi-audio::playback` owns stable opaque device locators, partial-failure discovery, exact channel,
sample-rate, sample-format, and callback-buffer capability ranges, configuration revalidation, and
stream lifecycle. CPAL 0.17.3 supplies the platform adapters while remaining compatible with the
workspace Rust declaration, and ringbuf 0.4.8 supplies the preallocated lock-free SPSC sample queue.
The producer accepts normalized interleaved `f32` only in whole frames. The platform callback enters
the audio execution domain, converts directly into every CPAL device sample type, substitutes
silence without blocking or growing storage, advances `AudioMasterClock` by every complete device
frame, and records only atomic telemetry. Linux CI now installs ALSA development headers through the
existing shared native-dependency provisioner.
