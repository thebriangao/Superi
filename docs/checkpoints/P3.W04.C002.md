# P3.W04.C002 timeline clip presentation

## Outcome

The existing editing timeline now presents every canonical clip as an accessible, immediate, identity-preserving button with its authored name, source state, exact timing, grouping, linking, target, synchronization, retime, marker, metadata, multicam, effect, driver, and automation evidence. Video clips progressively display real generated filmstrips or thumbnails, audio clips display real generated waveforms, and exact in-range clip-gain samples appear as clip-relative keyframe diamonds while nested, missing, offline, stale, unsupported, or failed preview states remain explicit.

## Architecture and integration

C001's `projectTimelineDocument` remains the sole strict canonical timeline parser and geometry owner. `timeline-clip-presentation.ts` is a read-only supplemental projection over that frozen model and the same complete `EditorStateSnapshot`; it correlates media and timeline sources, exact time maps, markers, metadata, multicam state, clip-scoped graph routes, parameter drivers, and attached clip-gain automation without reconstructing item timing or adding authored state.

Graph effect presentation starts at the canonical compiled clip node, follows outgoing edges through non-timeline processing nodes, and stops at the next timeline-owned node. This keeps track and output effects out of clip badges. The current compiled timeline graph cannot legally carry effects-owned animation curves, so visual keyframes are not inferred from arbitrary JSON; only exact clip-gain automation samples from the attached audio owner appear as keyframes.

The existing media-library and preview commands remain the only preview owners. The React surface reads one revision-matched library snapshot, deduplicates media identities, generates bundles sequentially, accepts only matching media and freshness identities, ignores stale asynchronous results, and never blocks the authored clip surface on derived media.

## Editorial intent and gestures

Canonical timeline selection remains a distinct authored state indicator. Clip activation routes a stable `superi.editor.state` reference through the existing shared application selection reducer, with Shift, Command, or Control extending selection and ordinary activation replacing it. Pointer activation, Enter, and Space operate the native clip button without moving the timeline playhead or creating a second selection or mutation owner.

Every clip retains exact source and record ranges, stable clip, track, timeline, and source identity, canonical and shared selection states, link and group peers, track targeting and synchronization, relink status, time-map rates, multicam intent, markers, metadata keys, graph effects and drivers, and sample-clock automation. Tooltips preserve the detailed evidence when narrow clips cannot display every badge.

## Failure handling

Malformed supplemental state produces an explicit clip-detail warning while the proven C001 canvas remains available. Missing graph state yields no fabricated effects. Missing media records, offline sources without a derived fallback, media-library revision changes, preview freshness changes, and command failures each retain truthful unavailable states, and cleanup prevents late results from replacing a newer project snapshot.

## Verification

The focused contract proves integration with the existing exact canvas projection, external global-start placement, mixed record clocks, stable relationships, relink state, retime state, markers, metadata, multicam state, clip-scoped effects and drivers, exact clip-gain sample keyframes and clip-relative positions, deep immutability, malformed-state rejection, real preview-command composition, and shared-selection routing. The complete frontend suite, strict TypeScript check, and production Vite build pass against the real React consumer; codebase-map and repository checkpoint verification are recorded in the checkpoint execution evidence.

## Remaining constraints

Preview generation remains asynchronous and intentionally bounded to one media identity at a time. Visual effect animation curves require a future legal persisted owner before the desktop can present them. Authored timeline mutation, effect editing, automation editing, playback transport, and render evaluation remain with their existing or future lower-domain command owners.
