# P1.W07.C009

## Outcome

Added a blocking Ubuntu 24.04 GitHub Actions job that runs the open-tree core workflows inside a Linux network namespace with outbound access disabled.

## Architecture and integration

The job prepares the pinned Rust toolchain, locked dependencies, workspace binaries, and test executables before isolation. Namespace entry explicitly retains the private libva header, pkg-config, native linker, and runtime linker paths plus the approved libvpx runtime path. `open/ci/run-network-isolated.sh` then requires a distinct network namespace, reads its namespace-aware interface inventory from `/proc/net/dev`, rejects non-loopback interfaces and IPv4 routes, verifies a numeric outbound socket cannot connect, forces Cargo offline, and runs the workspace tests, canonical fixture validation, and the headless CLI consumer. `open/ci/network-isolated-contract.sh` keeps the workflow, isolation boundary, provisioned media environment, namespace-aware probes, and core command set coupled through a repository-owned executable contract.

## Implementation process

The initial verified base was `a11cecd`, and the final implementation was reconciled with C002, C003, C005, C008, and the canonical vertical-slice contract. The workflow contract was added and run first, producing the expected missing-workflow failure, followed by the smallest workflow and harness implementation. Hosted red evidence from runs `29308007012`, `29379849198`, `29382902840`, and `29383899096` successively exposed the distribution libva API gap, missing native link paths, a host-mounted sysfs interface view, and dropped private libva paths at the privileged namespace boundary. The corrections retained the unchanged H.266 API requirement, one checksum-pinned libva 2.22 provisioner, and a strict namespace boundary while moving interface discovery to the namespace-aware procfs view and explicitly transferring the provisioned media environment through `sudo env`.

## Verification

The final focused shell, shared provisioning, namespace-probe contract, workflow YAML, map, boundary, and repository hygiene checks passed in the checkpoint-specific environment. Earlier widening proof also completed the locked workspace build, strict all-target and all-feature Clippy, documentation tests, fixture validation, and the complete local workspace test command; only three pre-existing macOS VideoToolbox and AudioConverter runtime tests were rejected by host frameworks. The final delivered Ubuntu run is the authoritative Linux namespace and offline core-command proof, while the earlier hosted failures remain durable red evidence.
