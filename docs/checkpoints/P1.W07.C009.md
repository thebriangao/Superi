# P1.W07.C009

## Outcome

Added a blocking Ubuntu 24.04 GitHub Actions job that runs the open-tree core workflows inside a Linux network namespace with outbound access disabled.

## Architecture and integration

The job prepares the pinned Rust toolchain, locked dependencies, workspace binaries, and test executables before isolation. `open/ci/run-network-isolated.sh` then requires a distinct network namespace, rejects non-loopback interfaces and IPv4 routes, verifies a numeric outbound socket cannot connect, forces Cargo offline, and runs the workspace tests, canonical fixture validation, and the headless CLI consumer. `open/ci/network-isolated-contract.sh` keeps the workflow, isolation boundary, and core command set coupled through a repository-owned executable contract.

## Implementation process

The initial verified base was `a11cecd`, and the final implementation was reconciled with C002, C005, and C008 on `b8f947f`. The workflow contract was added and run first, producing the expected missing-workflow failure, followed by the smallest workflow and harness implementation. Review corrections made path resolution independent of `CDPATH`, pinned checkout and disabled credential persistence, installed the declared toolchain and Linux media dependencies explicitly, built checksum-pinned libvpx 1.16 before isolation, and transferred only the required Rust and media environment into the privileged namespace.

## Verification

The final focused shell, workflow, map, boundary, and repository hygiene checks passed in the checkpoint-specific target environment. Earlier widening proof also completed the locked workspace build, strict all-target and all-feature Clippy, documentation tests, fixture validation, and the complete local workspace test command; only three pre-existing macOS VideoToolbox and AudioConverter runtime tests were rejected by host frameworks. The delivered Ubuntu run is the authoritative Linux namespace and offline core-command proof.
