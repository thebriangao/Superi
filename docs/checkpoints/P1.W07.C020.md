# P1.W07.C020

## Outcome

Created a deterministic timing baseline covering constant frame rate, variable frame rate,
drop-frame labels, a forward timestamp gap, and a timestamp reset. The versioned baseline contains
five cases and 18 samples in one strict, compact catalog with explicit continuity segments.

## Architecture and integration

`superi-fixture-tool` owns the dependency-free generator and `generate-timing` command. It emits a
fixed CRLF catalog and schema 1 manifest into a new directory, records exact provenance, byte count,
and SHA-256, and refuses every existing output path. `superi-media-io` consumes the checked-in
catalog through the public `PacketTiming`, `VariableFrameRateMap`, `TimestampNormalizer`, and
`TimecodeDescription` interfaces.

The catalog keeps decode order distinct from presentation order for VFR. It also treats timestamp
gaps and resets as explicit source segment boundaries: an unsegmented stream is invalid, while each
declared segment can be normalized to zero and restored without losing its source coordinates. The
drop-frame case follows Apple's documented 29.97 convention by skipping labels at the minute
boundary without dropping physical frames.

## Implementation process

The synchronized base was `036149c`. Generator, CLI, and real consumer contracts were written and
run first. The generator contract failed because the timing API did not exist, the CLI contract
failed because `generate-timing` was unsupported, and the media-I/O contract compiled and failed
because the canonical catalog was absent. The implementation then added the fixed timing table,
deterministic manifest generation, safe CLI behavior, canonical artifacts, strict parsing, and
public-interface proof.

Primary semantics were checked against Apple's Core Media sample timing definition and QuickTime
timecode documentation. Core Media distinguishes presentation timestamp, decode timestamp, and
duration, while QuickTime defines drop-frame as skipped timecode values rather than dropped video
frames and associates timecode samples with contiguous source segments.

## Verification

Focused generator tests prove two independent outputs match one another and the canonical files
byte for byte, with exact case and sample counts and preserved existing destinations. CLI tests
prove the success report, manifest creation, usage contract, and overwrite refusal. The media-I/O
contract proves fixed schema and inventory, CFR continuity, VFR decode-to-presentation ordering,
continuous physical drop-frame samples across skipped labels, rejection of unsegmented gaps and
resets, and reversible per-segment normalization.

The canonical fixture validator covers provenance, inventory, byte count, digest, safe paths, and
malformed manifest behavior. Broader fixture-tool, media-I/O, workspace, policy, formatting, and map
validation results are recorded in the ignored execution log.

## Delivery

This record is committed with the generator, CLI, canonical artifacts, consumer contract, fixture
documentation, and current codebase maps. The delivery commit containing this record is pushed to
`origin/main`; its SHA is recorded in the matching Google Docs completion description and ignored
execution log after it exists.

## Remaining constraints

These metadata fixtures do not claim encoded container round trips, hardware decoder behavior,
audio synchronization, color or HDR coverage, malformed media payloads, or the complete editorial
slice. Separate checkpoints own those baselines and may consume these timing cases through their
stable versioned identity.
