# P3.W03.C006

## Outcome

The project media inspector now lets editors persist bounded typed clip names, labels, ratings, keywords, comments, and favorite intent while showing current usage counts derived from authoritative timeline clip references.

## Architecture and integration

The C005 project-identity media-library snapshot remains authoritative. Each media item retains read-only `source_metadata`, generic `user_metadata`, and an adjacent typed `annotations` record; revision-fenced annotation replacement clones and validates the complete candidate before publishing one library revision, while reimport preserves the existing annotations and C005 state.

Usage is a read-only `MediaUsageIndicator` rebuilt from the current immutable project database by counting exact `ClipSource::Media` references across stable timeline identities. The usage field is skipped during serialization, so reopening or later timeline edits cannot retain stale derived counts.

The existing Tauri command registry, strict TypeScript bridge, and React media detail carry one authoritative replacement snapshot. Native validation bounds text, collection counts, rating values, duplicate terms, control characters, and total serialized annotation size without modifying media IDs, fingerprints, source paths, source metadata, generic user metadata, or bin placement.
