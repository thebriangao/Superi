# P2.W04.C006

## Outcome

Implemented prepared band-limited sample-rate conversion between independent source and device
sample clocks. Conversion preserves exact consumed and produced sample coordinates, ordered channel
meaning, anti-aliasing, and continuous ratio changes for bounded device-clock drift.

## Architecture and integration

`superi-audio::resample` owns a fixed-device-output converter prepared outside the callback with
all sinc state and planar scratch storage. The audio-domain call accepts a fixed maximum source
lookahead, reports the exact leading frames consumed, emits one fixed interleaved device block, and
retains source and device positions in their native `SampleTime` clocks. Positive device clock
error has an explicit sign convention and smoothly lowers the output-to-input ratio so a faster
device consumes more source frames without a block-edge pitch step.

## Implementation process

The public contract was written and observed failing before the conversion API existed. Research
selected the MIT-licensed rubato 0.16.2 asynchronous sinc converter because it supports Rust 1.80,
preallocated processing, fixed output blocks, filter-delay reporting, and ramped ratio adjustment.
Implementation then added strict configuration, complete prevalidation before DSP mutation,
prepared deinterleave and reinterleave storage, exact reports, bounded clock observations, and
actionable classified errors.

## Verification

Focused tests cover 44.1 to 48 kHz stereo channel identity and continuity, 96 to 48 kHz stop-band
suppression, exact multi-block source and device clock advancement, wrong-domain and discontinuity
rejection without position advance, and both device-drift signs. The final audit also runs the full
audio package, the engine manifest consumer, formatting, targeted warnings-denied lint, dependency
policy, map validation, and the checkpoint verifier within the user-directed 180-second timebox.

## Delivery

This record ships with the converter, public contract, pinned dependency, lockfile, and current
codebase maps. The delivered commit is pushed to `origin/main`, and its full SHA is recorded in the
paired checklist after remote containment is confirmed.

## Remaining constraints

The engine does not yet bind decoded media and timeline slices into the audio graph, and mixing,
routing execution, metering, plugin hosting, variable device callback aggregation, physical audio
hardware, underrun, and long-soak proof remain owned by later checkpoints and hardware lanes.
