# P2.W04.C010 Audio input capture

## Outcome

Superi audio now exposes production operating-system input discovery, exact configuration,
record arming, independent input monitoring, and bounded sample-accurate capture. Captured samples
retain their physical input clock and channel index through disarmed intervals and backpressure.

## Architecture and integration

`superi-audio::capture` owns the CPAL input boundary and a preallocated pair of lock-free rings.
The callback publishes channel-indexed recorded samples with exact `SampleTime` coordinates and a
separate normalized monitoring stream that can feed the existing bounded output producer. Atomic
arming and monitoring controls take effect at callback boundaries without callback allocation,
blocking, or locks.

## Implementation process

The public input contract was added first and failed because the capture module did not exist.
Implementation then added stable device locators, honest capability discovery, exact configuration
validation, typed CPAL callbacks, owning stream lifecycle, independent whole-frame backpressure,
telemetry, and bounded readers. The monitoring contract bridges captured input into the existing
output path rather than introducing a parallel playback abstraction.

## Verification

Focused contracts cover capability containment, arming, independent monitoring, exact timing
through gaps, whole-frame pressure, malformed input, audio-domain conflicts, stable locators, real
host discovery, thread transfer, and 512,000 frames without drift. The complete audio package,
real monitoring-to-output consumer, formatting, warnings-denied Clippy, documentation, map
validation, and changed-path checkpoint verification are delivery gates.

## Delivery

The delivered commit is recorded in the live Phase 2 descriptions tab after remote containment is
confirmed. The matching main-tab specification remains unhighlighted until fresh verification on
the delivered tree passes.

## Remaining constraints

Microphone permission, physical latency, hot-plug recovery, semantic speaker labels, and hardware
soak evidence remain platform and physical-test responsibilities. No engine owner yet binds
captured samples into timeline media, authored routing, clip processing, or the complete final mix.
