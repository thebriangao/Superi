# P3.W06.C010

## Outcome

Implemented configurable keyboard shortcuts for every registered application command, with
transactional conflict detection, explicit unbinding and reset behavior, deterministic JSON import
and export, accessible keyboard capture, and private cross-session restoration.

## Architecture and integration

The immutable `ApplicationRegistry` remains authoritative for command identities, titles, default
bindings, enablement, and execution. A separate schema 1 `KeyboardShortcutProfile` stores only user
overrides and explicit unbindings. The pure shortcut model canonicalizes portable modifiers, named
keys, and Unicode logical keys; rejects malformed or platform-specific bindings; detects conflicts
across the effective live command catalog; and reserves native File, Edit, and application menu
accelerators before any profile mutation. Unknown command overrides remain bounded, exportable, and
inactive so a temporary catalog difference does not silently erase user intent.

`ApplicationProvider` owns the accepted profile and resolves the global key adapter through it while
preserving the existing editable-target and IME composition boundaries. The System route includes a
dedicated Shortcuts panel with labeled read-only capture fields, current and default values, clear and
reset controls, two-step reset-all confirmation, conflict alerts, retained-command status, bounded
local JSON import, Blob export, native reservation disclosure, responsive layout, and visible focus.
The route sidebar consumes the same effective values as command dispatch.

Desktop-shell schema 3 persists the bounded profile beside presentation-only workspace state in the
existing private atomic session record. Schema 1 and 2 records migrate to default shortcuts, schema 3
validates profile bounds and duplicate command identities before live mutation, and a corrupt profile
restores a valid workspace with default shortcuts plus actionable failure state. Project bytes,
transaction history, engine state, media state, and timeline gestures remain with their existing
owners and never enter shortcut import, export, or persistence.

## Remaining constraints

Configurable bindings apply to registered global application commands. Timeline-local unmodified
editing gestures and native menu commands keep their specialized owners; global configurable
bindings require the portable primary modifier or Alt so those input domains remain predictable.
The main window remains the established publisher of private desktop presentation state, while every
window restores the latest native profile snapshot on startup.
