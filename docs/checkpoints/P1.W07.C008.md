# P1.W07.C008

## Outcome

The open workspace now has a deterministic static boundary scanner that rejects forbidden network
client dependencies and APIs, closed-product imports, and symlink escapes. The scanner is a normal
workspace member, so its canonical-tree contract participates in automated workspace tests.

## Architecture and integration

`superi-boundary-tool` is a dependency-free repository utility under `open/tools/`. It walks Cargo
manifests and Rust source without following symlinks or fetching dependencies, recognizes direct,
renamed, target, build, and development dependencies, lexes Rust while ignoring inert comments and
strings, and emits stable code, path, line, and message diagnostics. Its CLI provides the explicit
`cargo run -p superi-boundary-tool -- check .` contributor surface. Every cross-platform build job
executes the locked command before building, and a scanner contract requires one invocation per
declared build job. The boundary-tool module map plus workspace and global maps record ownership,
the public surface, integration flow, fresh proof, and remaining limits.
