# P1.W05.C011 color metadata propagation

## Outcome

Color source identity, exact embedded payloads, processing interpretation, ordered transform
history, monitoring intent, and delivery intent propagate without ambiguity from decoded media to
the viewport and export boundaries.

## Architecture and integration

The canonical `ColorPipelineMetadata` contract is owned by `superi-image`, below both media and
graph in the dependency tiers. `VideoFrame` initializes and validates it against the decoded
`VideoFormat`; graph and timeline retain it immutably; cache equality and hashing include the full
record; engine creates independent terminal display and output branches from cached scene state.
Graph does not depend on `superi-color`, and viewport metadata never mutates export metadata or the
shared cached scene result.

## Implementation process

The verified base was `a11cecdbf19ae1de90d94324abe844db49ed0c85`. Repository architecture,
image metadata, media frame, graph, timeline, cache, engine, and color-consumer contracts were read
before design was frozen. The end-to-end engine contract was written red, then the smallest required
lower-tier contracts replaced only scaffold modules and extended established frame metadata paths.

## Verification

The headless integration contract exercises media, graph, timeline, cache, viewport, and export in
one path. It verifies exact source tags and ICC bytes, semantic stage ordering, endpoint continuity,
cache separation, independent terminal branches, and rejection of a source/format mismatch. Fresh
proof covers all affected suites, the locked workspace suite with only the three documented macOS
hardware exclusions, all-feature checking, strict Clippy, warnings-denied rustdoc, formatting,
fixtures, dependency direction, open-boundary policy, Unicode dash checks, and all 24 codebase
maps. A post-integration Rust 1.80 check documented the synchronized lockfile's edition 2024
compatibility boundary before compilation of the five lower changed crates.

## Delivery

The implementation, tests, map updates, and this record are delivered together in one lowercase
three-sentence commit to `origin/main`. The exact delivered SHA is recorded by repository history
and the matching Google Docs completion description after remote containment is confirmed.

## Remaining constraints

This checkpoint intentionally does not implement color transform math, tone mapping, legal-range
conversion, GPU shaders, codec-specific tag parsing, or UI selection. Those responsibilities remain
with their dedicated checkpoints and existing color, codec, GPU, and UI ownership boundaries. The
synchronized workspace cannot compile with Rust 1.80 because locked `hashbrown` 0.17.1 requires
Cargo's edition 2024 feature. The complete workspace and every affected crate pass the current
stable toolchain.
