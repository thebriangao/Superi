# P3.W06.C003

## Outcome

Implemented native `.superi` project file association ingress for development builds. Startup
arguments and macOS operating-system open events now activate projects through the existing durable
desktop lifecycle while preserving complete replacement state and visible failures.

## Architecture and integration

Tauri declares one exact Superi Project association with editor role, owner rank, MIME type
`application/x-superi-project`, and exported type `com.superi.project` conforming to database and
data documents. The native association module accepts only case-insensitive `.superi` paths,
normalizes relative startup arguments, converts only local file URLs, canonicalizes existing files,
deduplicates in first-seen order, and ignores unrelated process arguments and nonfile resources.

Accepted paths run on Tauri's blocking runtime and enter
`DesktopProjectState::execute(DesktopProjectCommand::Open)`. That existing sole owner validates
through `LocalProjectHost`, updates active identity and recents only after success, retains the last
valid project and recent history after failure, and preserves every downstream project, transaction,
recovery, and document identity boundary. The main window is restored and focused after each batch.

One typed Tauri event carries the attempted path, source, complete replacement snapshot when
available, and actionable failure when opening fails. The always-mounted React application shell
registers its listener before its initial snapshot read, rejects stale replacement revisions,
publishes the accepted project snapshot to existing presentation subscribers, and refreshes the
editor through its established owner. Route, panel, selection, workspace, and other transient
application state are not reloaded or reset.

## Implementation process

Focused contracts first failed because the native association module and its configuration and
consumer wiring did not exist. The implementation added only association selection and dispatch,
the exact Tauri declaration, the typed event adapter, and the existing application-shell and
System-panel presentation subscriptions.
No second project owner, public API method, schema migration, single-instance service, release
installer, or new manual file-opening path was introduced.

## Verification

The native contract creates and reopens a real `.superi` database through the durable project owner,
proves ordered argument and file-URL filtering and deduplication, and proves that a later missing
project retains active identity and recent history with structured failure state. The application
contract freezes exact association metadata, startup and macOS event routing, blocking-worker
dispatch, lifecycle Open reuse, the stable Tauri event, and React consumption.

Strict TypeScript checking, the production frontend build and suite, native formatting and
all-target checks, map validation, and the repository checkpoint verifier are recorded in the local
execution evidence before delivery.

## Remaining constraints

The current application owns one active project, and this checkpoint preserves that established
model. Windows and Linux association launches enter through startup arguments, while macOS also
supports live resource-open events. Cross-process forwarding, installed default-handler mutation,

outside this checkpoint.
