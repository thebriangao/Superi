# P3.W02.C001

## Outcome

Implemented one native wgpu viewport in the production application window with direct,
color-managed presentation from a managed canonical GPU texture.

## Architecture and integration

`GpuDisplayPresenter` binds one explicit `OutputColorTransform` to the managed GPU device lifetime,
samples canonical premultiplied `Rgba16Float` storage, derives gamut and sRGB shader constants from
that reference, aspect-fits arbitrary source and surface extents, and retains the sampled source
through ordered submit and present completion. The presentation command never maps or reads back
frame pixels.

The Tauri shell creates one input-transparent native child window on the UI thread and transfers its
`NativeViewportSurface` to the sole `GpuSubmission` execution domain. That owner selects a
surface-compatible adapter, creates the device, resource manager, exclusive submission queue, and
one minimally cleared managed source, then serializes configure, acquire, encode, submit, present,
wait, and teardown. React owns only the editing-panel rectangle, scale, visibility, and returned
status; no media pixels cross the webview boundary.

## Implementation process

Implementation began after application prerequisites P3.W01.C005 and P3.W01.C006 were contained in
`origin/main`. The focused GPU display contract first failed because
`superi_color::gpu_display` did not exist, then passed after the smallest presenter and
managed-resource lifetime seam were added. The native owner and exactly one persistent
`EditingWorkspacePanel` hook were integrated without adding the C002 no-IPC policy scan or the
C003 viewer variants.

## Verification

The frozen GPU display contract passes both aspect-fit and explicit display-reference cases. The
focused native host gate passes checked physical placement and hidden zero-extent behavior. Strict
frontend TypeScript checking and the production Vite build pass with the native viewport hook, and
the codebase map validator passes after source-backed color, GPU, workspace, and index updates.

## Delivery

The presenter, native host, editing-panel integration, focused contracts, maps, and this record are
delivered together by the commit containing this file. The exact delivered commit is recorded in
the matching checklist completion description after `origin/main` containment is confirmed.

owns source, program, composite, and color viewer render-result bindings and variants.
