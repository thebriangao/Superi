# P3.W06.C005

## Outcome

Implemented shell-level discovery for GPU adapters, audio input and output devices, codec
declarations, and the actual local AI runtime state. The System panel exposes live, degraded,
unavailable, and retained state with actionable failures and a manual refresh. A bounded private
cache keeps last-known operational visibility across sessions without becoming project, workspace,
device, route, codec, or model authority.

## Architecture and integration

One Tauri-owned `DesktopCapabilityState` runs native discovery on a blocking worker and returns one
strict schema-1 snapshot. GPU rows come from `superi-gpu::GpuInstance` adapter snapshots. Audio rows
come only from `superi-audio` input and output enumeration and retain exact sample-rate ranges,
channel counts, formats, callback-buffer constraints, defaults, and explicit unknown channel
meaning. Codec rows are assembled by the engine registry and projected through
`MediaCapabilitiesApi`. `superi-ai` owns an honest local-only capability snapshot whose runtime is
unavailable and whose executable pipeline list is empty.

The owner persists only a bounded advisory snapshot under the existing private recovery root.
Strict validation rejects malformed, future, oversized, or unsafe cache content. A failed live
domain may retain last-known data with `retained` freshness while displaying the current failure,
and cache failure never blocks live discovery. The frontend adapter validates every field, rejects
unknown data, and deeply freezes the resulting replacement before the production System panel
renders it.

## Audio and application boundaries

Discovery never constructs, starts, plays, pauses, discards, or reconfigures an audio stream. It
does not select a route, create a ring buffer, alter a sample clock, guess semantic channel layout,
or modify audible state. Active timing, synchronization, routing, and continuity remain with the
existing audio, engine, project, and playback owners.

Capability refresh also does not open a media source or codec session, create a GPU device, load an
AI model, mutate an editable artifact, or change project, selection, panel, route, and workspace
state. The retained file is operational evidence only and cannot restore or override user intent.
