# P1.W05.C003

## Outcome

Added explicit working-to-display and working-to-deliverable RGB transforms with deterministic SDR, HLG, and PQ encoding while preserving image identity, alpha, metadata, and authoritative output color meaning.

## Architecture and integration

`OutputColorTransform` consumes canonical or promoted `WorkingImage` values, uses the existing `WideGamutTransform` and HDR transfer contracts, and returns premultiplied RGBA binary32 `Image` artifacts. The boundary intentionally emits full-range RGB; YUV matrices, legal-range packing, integer quantization, configured looks, and executable ICC profile transforms remain with their existing GPU, config, and ICC owners.

## Implementation process

The verified base was `217e9d48703bcfd4736d949aea510c94505071bc`. External contracts first proved that the output API was absent, then the scaffold was replaced with explicit target semantics, validated options, ordered alpha and color processing, and shared diagnostic failures.

## Verification

The focused contract covers display and delivery targets, canonical binary16 input, binary32 processing, primary conversion before transfer encoding, absolute PQ reference-white scaling, HLG and extended SDR behavior, metadata and window preservation, invalid configurations, source mismatch, physical PQ limits, determinism, and concurrency traits. Six focused tests passed on stable and Rust 1.80, all 56 color tests passed, strict Clippy, rustdoc, formatting, and diff checks passed, and the all-feature workspace passed with only the native-codec package excluded after its host frameworks reported three unrelated unavailable errors.

## Delivery

The implementation, tests, and this durable record are delivered together on `origin/main`. The exact delivered commit is recorded in the master checklist after remote containment is verified.

## Remaining constraints

This checkpoint does not invent a tone mapper or approximate ICC processing. View looks, profile execution, YUV conversion, legal-range encoding, and integer or GPU storage conversion remain explicit downstream stages.
