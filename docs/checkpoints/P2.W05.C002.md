# P2.W05.C002

## Outcome

Implemented a stable built-in catalog for transform, crop, opacity, blend, composite, Gaussian
blur, sharpen, radial distortion, chroma key, invert, and grade nodes. Every schema exposes typed
ports, typed editable and animatable parameters, deterministic behavior, per-region caching, ACEScg
color requirements, exact version `1.0.0`, and the production capability requirement
`superi.render.gpu`.

The same catalog instantiates into any `EditableGraph<GraphValue<T>>` with caller-owned node, port,
and parameter identities. Compiled timeline graphs now use `GraphValue<TimelineGraphValue>`, so
ordinary effect state and exact editorial domain state coexist in one graph without reversing the
timeline-to-graph or effects-to-graph dependency direction.

## Shared editable state

`superi-graph::value` owns the neutral processing payload seam. `FiniteF64` rejects NaN and
infinity, preserves exact binary64 bits including signed zero, and validates those bits during
deserialization. `GraphValue<T>` retains exact caller state in `Domain(T)` beside scalar, vector,
color, matrix, Boolean, and bounded choice variants with checked constructors and typed accessors.

Direct parameter links clone every exact variant. Bounded graph expressions accept and produce only
the scalar variant, so colors, matrices, choices, Booleans, and domain values cannot be silently
coerced. Timeline compilation wraps every existing `TimelineGraphValue` in `Domain`, preserving its
stable schema, parameter identity, provenance, and semantic payload while opening the same graph to
processing nodes.

## Catalog and reference integration

`superi-effects::catalog` owns `EffectNodeKind`, stable families, exact schema construction,
deterministic discovery, and one atomic registry transaction. `superi-effects::authoring` accepts a
complete `EffectNodeBindings` value and supplies typed defaults without inventing stable instance
identities. Incomplete, duplicate, unknown, cross-direction, or mistyped bindings fail before a
node can enter editable graph state.

`superi-effects::reference` is a bounded CPU oracle for headless verification and future GPU parity,
not an engine playback or export path. `compile_reference_node` resolves literal, linked, and
expression-driven parameters from one immutable `GraphSnapshot`, requires the exact catalog schema,
binds incoming edges by semantic destination port, validates typed operation state, and produces a
`ReferenceEffectNode` that uses the existing `EvaluateNode<Image>` and `IntrospectNode` contracts.
Its state fingerprint covers the exact schema, resolved result-affecting values, discrete choices,
and semantic input-port bindings. Graph topology and upstream cache lineage remain graph-owned.

Reference inputs are restricted to unqualified premultiplied RGBA ACEScg in `Rgba16Float` or
`Rgba32Float`. Outputs preserve color tags, channel identity, metadata, sample representation, and
display window. Extended scene-linear RGB remains unclamped, while nonfinite samples and alpha
outside zero through one fail actionably. Final and temporary allocations, Gaussian kernels,
dimensions, channels, coordinate arithmetic, and loops are checked against caller-selected
`ImageLimits` before work proceeds.

## Operation semantics

- Transform uses the repository image transform with an explicit forward matrix, inverse-mapped
  pixel centers, nearest or premultiplied bilinear reconstruction, and transparent-black edges.
- Crop applies nonnegative pixel insets to the source data window and returns the requested region
  with transparent black outside the authored crop.
- Opacity scales premultiplied RGB and alpha together.
- Blend implements normal, multiply, screen, overlay, darken, and lighten with the W3C separable
  blend equation followed by source-over at explicit source opacity.
- Composite implements source-over, destination-over, source-in, source-out, source-atop, and xor
  through explicit Porter-Duff factors over premultiplied RGBA.
- Gaussian blur uses a normalized one-dimensional kernel in each axis with integer support
  `ceil(3 * sigma)`. Sharpen applies `source + amount * (source - blurred)` to premultiplied RGB and
  preserves source alpha.
- Radial distortion maps through `1 + k1*r^2 + k2*r^4`, requires the radial derivative to remain
  positive across the requested extent, requests a conservative source circle, and samples
  bilinearly.
- Chroma key measures straight ACEScg RGB distance, uses a deterministic smooth transition,
  suppresses spill along the normalized key direction, and reassociates the retained pixel.
- Invert interpolates straight RGB toward `1 - RGB`. Grade applies per-channel gain and offset in
  straight RGB. Both preserve alpha and reassociate their result.

ROI dependencies are state-derived. Blur and sharpen expand by their exact three-sigma support,
transform inverse-maps the requested bounds and includes bilinear support, radial distortion uses
its validated conservative circle, and local operations request the same region from each semantic
input.

## Research basis

The algebra was checked against W3C Compositing and Blending Level 1 and W3C Filter Effects Module
Level 1. The radial polynomial and monotonic physical-mapping requirement were checked against the
OpenCV camera-calibration and calibration model documentation. The Rust implementation is original
repository code and adds no third-party runtime implementation or license.

## Verification

`value_contract` proves finite exact bits, validated serialization, every neutral variant, typed
accessors, scalar expressions, and forbidden coercion. `compile_contract` proves all timeline
domain values remain exact and that one compiled timeline graph accepts shared processing state.

`catalog_contract` proves the complete kind set, exact schemas, stable order, families, ports,
animatable parameters, GPU capability, typed defaults, caller-owned identities, atomic registration
and graph publication, and invalid binding rejection. `reference_contract` exercises every
operation category with real binary16 and binary32 pixels, extended RGB, metadata retention,
premultiplied algebra, ROI, monotonic distortion, unsupported image meaning, invalid state, final
allocation limits, temporary working-memory limits, and fail-closed binary16 overflow.

`graph_workflow_contract` builds one real editable source, scalar control, expression-driven opacity
node, and typed edge. It compiles immutable old and new revisions through
`GraphEvaluationSnapshot`, evaluates actual pixels through the shared headless path, observes the
same diagnostic result, proves a direct control edit changes effect state and cache identity, and
proves the old reader remains unchanged. It also proves a same-name unsupported schema version
cannot enter the reference runtime.

Affected graph, image, timeline, and effects package suites, warnings-denied Clippy, formatting,
rustdoc, dependency and offline boundaries, codebase maps, and the full repository verifier are
recorded in the ignored execution log.

## Remaining constraints

The production schemas require GPU rendering, but this checkpoint does not add engine node
registration, GPU shaders, playback, viewport, export, project persistence, UI, keyframe curves,
masks, transitions beyond explicit compositing, text, tracking, or OFX hosting. The CPU module is a
deterministic reference and headless contract only. Production GPU evaluation and end-to-end
timeline playback remain later integration work.
