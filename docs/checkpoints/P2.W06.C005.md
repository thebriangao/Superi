# P2.W06.C005

## Outcome

Implemented one render and export transaction across prepared source reads, decode, immutable graph
evaluation, delivery color, audio processing, deterministic encoder selection, and complete
elementary packet publication. The engine retains exact source, graph, color, alpha, audio, timing,
metadata, and backend-selection evidence so a caller can inspect what was rendered without mistaking
elementary streams for a muxed container.

Normal completion requires every selected decoder and encoder to reach end of stream, every output
packet to retain nonempty bytes and the configured stream and timebase, and the exact union of
encoded timing to equal the exact union of submitted input timing. No artifact becomes visible until
those checks and a fresh lifecycle permit check pass.

## Architecture and integration

`RenderExportRequest` binds one exact source seek position, one video route and or one audio route,
the caller's registered-fallback policy, and a revision-scoped export lifecycle permit. Each
`ExportStreamRoute` carries the source stream identity and complete `EncoderConfig`. The engine
selects one encoder from the ordinary backend registry, records the selected backend and fallback
evidence, and never retries another factory after selection.

`ExportMediaSession` is the codec-neutral source and decoder seam. Its production implementation for
`AcquiredMediaSource` consumes the source and decoder owners prepared by `TimelineResources`, while
the trait also permits deterministic contract sessions. The transaction resets each decoder and
encoder, seeks the source exactly, accepts only complete reads, routes packets by source stream,
drains decoder output before and after flush, drains encoder output before and after flush, and
resets the completed transaction before publication.

`HeadlessGraphVideoRenderer` binds each decoded `VideoFrame` through
`SharedExportDecodedFrame`, evaluates the ordinary immutable `GraphEvaluationSnapshot`, and returns
the existing `PlaybackSceneFrame<V>` semantic envelope with its graph revision. The decoded binding
shares frame ownership without copying pixel storage and is cleared immediately after each graph
evaluation. `ExportVideoDelivery<V>` owns the distinct delivery-color and output-alpha stage, while
`ExportAudioGraph` owns prepared audio processing and records its `AudioGraphId`.

`RenderedExportVideo<V>` and `RenderedExportAudio` preserve the exact values submitted to the
encoders. `ExportVideoProvenance` retains decoded frame semantics, graph revision, scene format,
scene color history and alpha, delivery format, delivery color history and alpha, timestamp, and
duration. The encoder-facing video format must name the terminal delivery color space, while the
complete source-to-delivery history remains separate provenance. `ExportAudioProvenance` retains
graph identity, complete audio format, typed metadata,
sample timestamp, and duration. `ElementaryPacketStream` retains the encoder configuration,
selection evidence, inputs, and complete packets, and `RenderExportArtifact` groups those streams
with the authoritative source identity.

## Normal, degraded, and recovery behavior

Normal video processing requires decoded timing to match graph timing, nonterminal graph color
history to extend the decoded history, and the delivery branch to append a valid terminal output
stage without changing the declared scene meaning. Output format, sample format, channel layout,
pixel precision, alpha association, metadata, stream identity, timebase, payload, timing coverage,
and graph identity are validated before publication.

Partial or corrupt source reads, missing stage owners, stale lifecycle permits, cancelled operations,
wrong decoder output kinds, malformed graph or delivery semantics, incomplete packet coverage,
metadata loss, and premature codec end of stream fail without publishing an artifact. A selected
encoder factory failure is returned directly, even when another registered fallback exists. After
any in-transaction failure, the engine uses a fresh export operation context to reset every created
encoder and selected decoder so later work can recover independently of a cancelled caller context.

## Verification

The integration contract was written first and failed because the render and export surface did not
exist. Its paired deterministic path proves one video and audio transaction through the real graph
snapshot, decoded-frame binding, delivery branch, audio stage, lifecycle permit, registry selection,
codec drains, exact semantic provenance, complete packet publication, and reset lifecycle.

The same contract proves fallback disallow behavior, selected-factory failure without exception
retry, rendering-subsystem degradation, stale permit rejection, corrupt partial-source recovery, and
later success through a fresh transaction. A production acquired WAVE path opens the canonical
fixture, uses the in-tree PCM decoder, processes the block through the audio stage, uses the in-tree
PCM encoder, and preserves all 4,410 stereo samples exactly. A production acquired WebM path opens
and decodes all 96 AV1 frames, evaluates the shared graph and delivery branch, and proves that the
libvpx VP9 encoder's duration rounding is rejected as degraded timing drift instead of publishing an
inexact artifact.

Focused, package, subsystem, dependency, map, documentation, and repository verifier evidence is
recorded in the ignored checkpoint execution log.

## Remaining constraints

The artifact owns complete elementary packet streams in memory. Container muxing, filesystem or
network publication, progress streaming, arbitrary stream counts, timeline range scheduling,
transport control, native GPU readback, and application or public API integration remain separate
owners. The generic graph and delivery values can remain GPU-resident when a caller supplies that
path, but this checkpoint does not claim a native GPU export sink or container output.
