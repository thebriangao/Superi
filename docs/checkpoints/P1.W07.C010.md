# P1.W07.C010

## Outcome

Created deterministic, read-only golden-test harnesses for exact frame output, exact audio output,
canonical timeline documents, and canonical project documents. Added versioned workspace fixtures so
all four harnesses run against committed expected results rather than self-generated test baselines.

## Architecture and integration

The harness lives in `superi-fixture-tool`, the repository utility that already owns shared fixture
policy, so runtime crates and the downward-only engine dependency graph remain unchanged. Frame
goldens bind payload bytes to layout, pixel, channel, color, and alpha semantics. Audio goldens bind
payload bytes to sample-clock origin, rate, frame count, channel order, sample encoding, and storage
layout. Timeline and project goldens recursively sort object keys while preserving arrays and exact
JSON scalar values.

## Implementation process

The focused contract was written and observed failing before the public golden module existed. The
implementation then added typed versioned envelopes, structural validation, canonical encoding,
read-only comparison, first-difference diagnostics, and expected and actual SHA-256 evidence. A
review found that temporary expected files were too self-referential, so the proof was strengthened
with independently committed, provenance-validated fixtures under `test-fixtures/golden/harness/v1`.

## Verification

Focused golden contracts passed with five tests covering exact frame and audio drift, sample timing,
channel meaning, canonical nested JSON, missing and invalid goldens, no-write behavior, and all four
workspace fixtures. The canonical validator accepted 11 fixture versions and 27 payloads; formatting,
focused strict Clippy, full all-feature workspace strict Clippy, and the workspace test ring excluding
only three named unavailable macOS native codec roundtrips passed. The unfiltered workspace attempt
reached only those same documented physical-lane failures with OS statuses -12908 and 1718449215.

## Remaining constraints

Audio, timeline, project, graph serialization, and render orchestration are still skeleton runtime
consumers, so this checkpoint supplies stable harness contracts without inventing their future domain
models. Golden verification intentionally has no automatic update or bless path; reviewed semantic
changes must create a new immutable fixture version.
