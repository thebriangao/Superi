# P1.W07.C021

## Outcome

Created a deterministic color and image-sequence baseline covering SDR, wide-gamut, PQ and HLG HDR,
straight and premultiplied alpha, 16-bit integer, half-float, and full-float precision. The versioned
baseline contains eight 2 by 2 images and a three-frame ACEScg editorial sequence in 448 payload
bytes.

## Architecture and integration

`superi-fixture-tool` owns the dependency-free generator and `generate-color` command. It emits two
fixed CRLF catalogs, one little-endian binary sample payload, and a schema 1 manifest into a new
directory. The manifest records exact provenance, byte counts, and SHA-256 digests, and generation
refuses every existing output path.

`superi-color` consumes the checked-in image catalog through public `Image`, `InputColorTransform`,
and `OutputColorTransform` interfaces. The transform proof preserves declared source intent,
unassociates alpha before transfer decoding, transforms in scene-linear ACEScg, reassociates alpha,
and emits the declared display or deliverable intent with authoritative color tags.

`superi-media-io` consumes the checked-in sequence catalog through `ImageSequenceSource`. Logical
images 0, 1, and 2 map to noncontiguous file frame numbers -2, 0, and 2 while preserving presentation
timestamps 48, 49, and 50 at 24000/1001 fps. Random access and seeking return the exact referenced
full-float payload bytes.

## Implementation process

The synchronized base was `19453e5`. Generator, CLI, color-consumer, and sequence-consumer contracts
were written and run first. The generator contract failed because the color API did not exist, the
CLI contract failed because `generate-color` was unsupported, and both real consumer contracts
compiled and failed because the canonical artifacts were absent.

The fixed cases follow BT.2100 PQ and HLG transfer meaning, BT.2111 test-pattern guidance, OpenColorIO
ACEScg and display-view roles, and OpenEXR half-float representation guidance. The implementation
keeps those semantics explicit in catalog fields and exact sample bits rather than relying on file
extensions or inherited display configuration.

## Verification

Focused generator tests prove two independent outputs match one another and the canonical files byte
for byte, with exact image and sequence counts and preserved existing destinations. CLI tests prove
the success report, complete usage contract, and overwrite refusal. The color contract proves SDR
transfer order, PQ reference-white normalization, HLG scene meaning, zero-alpha behavior,
wide-gamut round trips, exact half-float bits, and exact full-float bits.

The media-I/O contract proves strict cross-catalog references, per-image hashes, exact rational
timing, noncontiguous file numbering, random access, seeking, and unmodified frame payloads. The
canonical fixture validator covers provenance, inventory, byte count, digest, safe paths, and
malformed manifest behavior. Broader verification results are recorded in the ignored execution log.

## Delivery

This record is committed with the generator, CLI, canonical artifacts, consumer contracts, fixture
documentation, and current codebase maps. The delivery commit containing this record is pushed to
`origin/main`; its SHA is recorded in the matching Google Docs completion description and ignored
execution log after it exists.

## Remaining constraints

This checkpoint provides source fixtures and semantic consumer proofs. It does not define rendered
golden images, perceptual comparison tolerances, codec or container round trips, hardware decoder
behavior, malformed media, or the complete editorial slice. Separate checkpoints own those outputs
and may consume this baseline through its stable versioned identity.
