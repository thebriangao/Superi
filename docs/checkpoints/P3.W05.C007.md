# P3.W05.C007

## Outcome

Implemented image, alpha, red, green, blue, luminance, false-color, and clipping inspection for every native viewer role on the existing GPU-resident presentation path.

## Architecture and integration

One frozen shell catalog owns ephemeral per-viewer selection, while the strict placement DTO carries only the selected code beside existing geometry and visibility. The native viewport maps that code exhaustively into `GpuDisplayView`, queues it through the sole GPU submission owner, and reports selected and last-presented state separately so a queued request is never described as an already displayed frame.

`GpuDisplayPresenter` samples the unchanged canonical premultiplied RGBA16F texture. Image, alpha, channel, luminance, and false-color inspection operate on unassociated scene-linear values before the existing primary, gamut, transfer, and attachment stages; clipping classifies the configured display-linear result before transfer encoding and attachment clamping, and the native slice continues to reject non-none tone mapping. The compatibility constructor remains image view, diagnostic views are opaque, source CIE Y comes from the actual source primaries, and the fixed false-color thresholds and palette are deterministic Superi inspection semantics rather than an external-standard claim.

## Implementation process

Focused shell, native, and color contracts were added first and failed on the missing catalog, DTO, enum, and presenter APIs. The implementation then connected the real React consumer, native command owner, production shader, selected-versus-presented diagnostics, and all four existing viewer roles without adding frame IPC, readback, a renderer, authored state, or a second GPU owner.

The production presenter rendered all eight modes offscreen from quantized RGBA16F input and matched a CPU reference per pixel and channel. After exact lockfile dependency restoration, TypeScript checking, the Vite production build, all 86 frontend contracts, and all 13 desktop library tests passed; Vite reported only the existing dynamic-import and chunk-size warnings.
