# P2.W04.C014 native audio plugin resilience

## Outcome

Implemented deterministic native audio plugin discovery, bounded validation, exact state
persistence, worker fault containment, recovery and quarantine policy, and graph-wide plugin delay
compensation. Projects retain distinct state for each authored audio node, preserve exact native
bytes across save and reopen, and can restore the same component after a compatible installed
version upgrade without treating runtime readiness as authored project meaning.

## Architecture and integration

`superi-audio::plugins` owns format-neutral Audio Unit and VST3 identity, an integrity-checked
bounded state envelope, fixed worker and native latency evidence, the nonblocking isolated process
bridge contract, delayed-dry fault fallback, and callback-safe telemetry. The ordinary
`AudioProcessor` contract now reports immutable prepared latency. `AudioGraph::prepare` computes
cumulative path arrival, allocates route-local delay rings before callback entry, and aligns direct,
send, auxiliary return, submix, and master routes without changing authored topology.

The delivered native hosts supply their exact format state. VST3 uses bounded seekable `IBStream`
objects and follows component restore, controller component restore, and controller restore order;
capture retains independent component and controller bytes. Audio Unit capture and restore use the
class-info property through bounded binary Core Foundation property-list bytes. Both hosts report
native algorithmic latency to the prepared graph.

`superi-engine::audio_plugins` owns recursive VST3 package discovery, explicit Audio Unit candidate
ingestion, exact descriptor validation, a required restartable worker-process contract, activation,
checkpointing, fault observation, restart, quarantine, and project record conversion. Blocking
scan, lifecycle, bridge preparation, and state work require the background domain. Runtime fault
observation is serialized on EngineControl. Project records use the existing schema-4 extension
store and a stable audio-node instance identity, so multiple uses of the same component cannot
overwrite one another and unknown opaque bytes continue through existing save, copy, backup,
autosave, recovery, history, and migration owners.

## State and interruption behavior

The state schema records exact format, component identity, saved vendor and version evidence,
sample rate, native latency, fixed transport latency, independent component and controller payloads,
checked lengths, and SHA-256 integrity. A compatible upgrade matches the stable format-specific
component identifier while retaining the saved version as evidence. Successful capture replaces it
with the newly scanned identity. Truncation, trailing bytes, bad magic, bad digest, unsupported
schema, invalid identity, oversized native state, changed latency, and mismatched project instance
identity fail before publication.

An isolated prepared processor advances a matching delayed dry path on every block. Missing,
faulted, failed, or nonfinite worker output therefore stays on the same sample coordinate instead of
jumping ahead to current input. The processor reports fixed transport plus native latency, and the
graph compensates every faster converging route to the latest arrival. Worker failure is retained in
atomics for EngineControl observation and never aborts the audio callback or publishes partial wet
output.

## Verification

Test-first contracts cover exact state round trip and corruption rejection, compatible version
restoration, distinct per-node project records, deterministic healthy and malformed VST3 discovery,
background-domain enforcement, in-process worker rejection, activation timeout containment,
restart from the retained checkpoint, quarantine policy surfaces, real schema-4 database reopen,
fixed-latency dry fallback, parallel-path and auxiliary-return compensation, irregular block
partition equivalence, and preparation-time overflow rejection. The existing VST3 fixture now
performs real component and controller state traffic, and the macOS Audio Unit contract captures and
restores Apple's Peak Limiter class information while retaining its native latency.

Delivery gates cover the complete audio, project, engine, and graph package suites, strict Clippy,
documentation, formatting, dependency and boundary policy, unsafe inventory, codebase maps, and the
full repository checkpoint verifier.

## Remaining constraints

Concrete platform IPC, shared-memory rings, sandbox launchers, Audio Unit registry enumeration, and
worker heartbeat or kill mechanisms remain platform application responsibilities behind the
delivered required process contract. Dynamic native latency notifications require a later prepared
graph rebuild owner; this checkpoint fixes latency for one prepared generation. Instruments, MIDI,
sidechains, plugin UI, broader parameter automation, decoded timeline sample binding, physical
device latency calibration, third-party compatibility coverage, and long-running hardware soak
remain separate work and platform evidence.
