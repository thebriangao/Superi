# P3.W06.C009

## Outcome

Implemented one searchable command palette for registered application commands and native desktop
actions. Every listed action exposes a stable automation identity, category, keywords, current
availability, disabled reason, and typed invocation path. The palette is transient, restores prior
focus when dismissed, and does not alter workspace continuity or persist search state.

## Architecture and integration

The application registry remains the authority for routes, panel visibility, layout reset and undo,
shared selection, and the palette opener. Registry definitions now require immutable discoverability metadata and expose
one pure availability lookup beside the existing typed command executor. The always-mounted
application consumer composes those definitions with a bounded desktop action projection for file,
history, recent-project, and quit behavior. Project mutations still enter the existing desktop
project owner or generated public API path, and the catalog has no transport or public API binding.

The pure command catalog rejects duplicate identities, freezes every descriptor, derives recent
project identities from encoded paths instead of mutable menu positions, performs token-complete
locale-independent ranking, and delegates only an application command identity or a typed desktop
shell intent. Current busy, active-project, undo, and redo state produces explicit disabled reasons.
Application route and panel controls now use the same registered command identities as keyboard and
palette execution.

One accessible modal React consumer owns only its query, highlighted result, pending action, and
failure presentation. It supports keyboard navigation, search, dismissal, focus restoration,
disabled explanations, stable identities, and shortcuts without entering the persisted workspace
presentation. An isolated stylesheet provides the dark desktop surface without changing shared
application styling.

The native Edit menu adds `Find Command...` with fixed `CmdOrCtrl+Shift+P` behavior. Rust maps the
stable menu identity to `OpenCommandPalette`, serializes it as `open_command_palette`, and delivers
it through the existing focused-webview shell event. The frontend routes that intent through the
registered palette opener, while native workspace intents route through registered application
commands.
