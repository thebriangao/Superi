# P1.W07.C015

## Outcome

Created an offline workspace tool that turns explicit platform-lane evidence into deterministic,
schema-versioned JSON reports. Reports expose performance regressions, golden mismatches, flaky
tests, and platform gaps while preserving attempts, reasons, environment context, and artifact links.

## Architecture and integration

`open/tools/superi-test-report` is repository tooling and remains outside the runtime crate DAG. Its
strict input boundary covers the result context required by `docs/platform-testing.md`; the library
validates and canonicalizes evidence, derives status and findings, and the CLI writes a report before
returning a blocking failure. Existing image-reference comparisons, diagnostic counters, fixture
revisions, platform capability discovery, and external raw artifacts can feed this boundary without
adding an upward runtime dependency.

## Implementation process

Verified base `a11cecd` already contained the platform matrix and canonical fixture policy. Added
failing library contracts first, implemented deterministic derivation and validation, then added a
failing CLI workflow contract before wiring safe non-overwriting output and exit behavior. A shared
disk resource gate removed generated output from the worktree and assigned remaining proof to one
checkpoint-specific target under `/private/tmp`.

## Verification

Focused contracts cover all four finding categories, contradictory evidence rejection, automatic
missing-suite gaps, attempt retention, canonical repeatability, blocking output creation, and
non-overwrite safety. Final workspace, all-feature, offline, Rust 1.80, formatting, Clippy, doc-test,
fixture, and real CLI workflow results are recorded in the ignored execution log for this checkpoint.

## Delivery

The implementation, tests, report contract, matrix integration, and this durable record are delivered
together in the commit containing this file. The delivered branch is `origin/main`; the exact commit
is resolved with `git log -1 --format=%H -- docs/checkpoints/P1.W07.C015.md` after delivery.

## Remaining constraints

The tool does not execute test suites, upload artifacts, invent unavailable hardware evidence, or
parse unstable test-harness output. CI and hardware-lane checkpoints remain responsible for producing
complete input evidence and retaining referenced raw artifacts; missing or unimplemented coverage is
intentionally reported as a gap.
