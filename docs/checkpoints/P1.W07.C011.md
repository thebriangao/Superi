# P1.W07.C011

## Outcome

Created a stable Cargo benchmark harness for decode, graph evaluation, upload, playback, cache, render, and project save/load, with a real public graph evaluator workload.

## Architecture and integration

`superi-bench` is a repository tool outside the runtime crate dependency DAG. It provides canonical stage identities, bounded warmup and sampling, real workload registration with fixture identity, deterministic integer statistics, complete environment context, stage filtering, and schema-versioned JSON output. Its shipped graph workload builds a deterministic three-node DAG and pulls it through `superi_graph::eval::LazyEvaluator`; unregistered consumer paths remain explicit gaps and never become fabricated performance results.

## Implementation process

The verified initial base was `a11cecd`, and the preserved work was rebased onto synchronized base `6f69a57` after the repository added substantive graph evaluation. Initial contracts produced the expected absent benchmark-target failure, and a second focused contract produced the expected missing graph-workload failure before the public evaluator consumer was registered. Official Cargo and standard-library documentation grounded the stable `harness = false` target and `std::hint::black_box` optimization barrier.

## Verification

Eight focused contracts cover stable stage order, bounded configuration, complete context, warmup exclusion, invocation counts, deterministic percentiles, filtering, duplicate rejection, JSON escaping, explicit gaps, workload failures, and the real lazy graph evaluator path. Fresh proof passed the optimized complete-context runner, strict focused and all-feature workspace Clippy, 24-module map validation, dependency and open-boundary checks, all fixture validation, documentation tests, and all-target workspace tests outside the native platform-codec package. The complete workspace test was also attempted and reached only three macOS VideoToolbox and AudioConverter host-framework availability failures outside this tool.

## Remaining constraints

Only real subsystem consumer paths may register workloads. Decode and upload have substantive lower-level contracts but no bounded benchmark consumer and versioned workload fixture yet; playback, cache, render, and project persistence remain absent, so all six stay visible as gaps. Incomplete environment context remains diagnostic rather than accepted performance evidence.
