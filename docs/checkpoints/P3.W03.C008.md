# P3.W03.C008

## Outcome

The project media inspector now owns durable replaceable proxy and optimized-media attachments, inspectable lifecycle status, explicit quality selection, and transparent deterministic fallback to the authoritative source.

## Architecture and integration

C008 extends each C007 `MediaBrowserItem` beside unchanged source identity, annotations, usage, selections, and tracked regions. Derived attachments bind purpose, quality, status, source fingerprint, source revision, byte length, and artifact identity without replacing the imported source.

One revision-fenced command atomically creates, replaces, removes, or selects derived media. Resolution accepts only ready exact-purpose, exact-quality, source-fresh attachments and otherwise reports an explicit original-source fallback.

The strict TypeScript bridge and existing React media detail expose attachment status and original, proxy, or optimized choice. The existing engine cache, generation, and proxy-substitution contracts remain the lower execution authority.

The smallest implementation then extended the current native presentation store, Tauri registration, typed bridge, and selected-media consumer without adding C009 relink, replace, conform, offline, or later search behavior.
