# P3.W06.C015

## Outcome

Implemented one shared dockable Inspector panel on every production workspace route. The panel
projects current workspace and selection intent, public project metadata, engine-owned history,
exact playback state, editor failures, and bounded operational notices into four stable semantic
groups. Engine readiness, refresh, detachment, pending commands, degradation, and failure remain
explicit without replacing the authoritative System owner.

## Architecture and integration

`app/src/application-inspector.ts` is the pure presentation boundary. It accepts only the current
application presentation, last-valid public editor snapshot, project history evidence, playback
observation, command failure, and bounded notification state, then returns deeply frozen Inspector,
Metadata, History, and Diagnostics groups. It creates no React, generated API, transport, Tauri,
project, history, persistence, polling, or diagnostic ownership.

`app/src/application-inspector-panel.tsx` is the real registry-backed consumer. Existing dock
continuity retains its route placement, visibility, active tab, and focus. Refresh delegates to the
current editor snapshot owner, recovery navigation opens System, and clearing transient shared
selection captures the exact prior selection and anchor for one precise restore. Isolated responsive
CSS consumes only the fixed theme's semantic chrome tokens and does not alter native viewer or
authored marker color-data meaning.

## Implementation process

The focused test first failed because the pure projector did not exist. Implementation added the
four-group model, honest engine classification, bounded safe notices, the mounted all-route panel,
stable toggle command, existing-owner actions, exact selection reversal, responsive semantic
styling, and source contracts excluding direct API and Tauri access. The C012 theme delivery was
integrated before source work so the panel uses the current color-critical application contract.

## Verification

The focused inspector contract passes three tests covering exact group order and values, project
identity and history, deep immutability, unchanged input, detached, pending, degraded, and failed
engine states, all-route dock registration, precise action wiring, exact selection restoration, and
transport exclusion. The combined inspector and application-presentation ring passes 11 tests.
Under pinned Node 24.13.0, strict TypeScript passes, the production Vite build transforms 92 modules
and emits the hashed application bundle, and the complete frontend suite passes 160 tests.

## Remaining constraints

The inspector intentionally retains only bounded process-local application notices; cross-session
operational evidence remains with the existing crash, workspace, project, lifecycle, and native
owners. Authoritative lifecycle and recovery controls remain in System, native project history
remains session-owned, and the panel does not add background polling, arbitrary metadata editing,
durable diagnostic storage, or direct engine control.
