# P3.W04.C001 timeline canvas foundation

## Outcome

The editing workspace now renders the canonical public timeline document as a precise interactive canvas with sticky track headers, an adaptive ruler, record-positioned clips and transitions, a frame-snapped playhead, explicit in and out range, native scrolling, and pointer-anchored zoom. The surface remains dependable across empty, short, and complex timelines because bounds, tick density, visible geometry, and gesture results are deterministic.

## Architecture and integration

`ApplicationProvider` remains the sole public editor-state presentation owner. `TimelineWorkspace` receives its existing immutable canonical timeline document and playback observation, while `timeline-workspace.ts` strictly validates revision 1 envelopes and projects exact rational record and source ranges into a frozen display model. The projection retains stable track, item, media, nested-timeline, group, link, transition, selection, target, and synchronization identities instead of inferring replacement editorial state.

Canonical record positions remain exact local ranges while display coordinates apply the timeline's external global start, so hour-based program timecode does not create empty canvas space or rewrite record evidence. Stored bottom-to-top track order is presented topmost first, and one visible-span overscan bounds mounted item content during zoomed navigation without changing the immutable model.

The canvas owns only transient viewing intent: playhead, range, scale, and scroll. Click and drag gestures snap to the canonical frame rate, keyboard steps advance exact frames, range handles stay ordered, native scroll remains available, Shift plus wheel pans horizontally, and Command or Control plus wheel zooms around the pointer. A fit action derives scale from the current timeline extent and viewport without mutating authored resources.

Authored timeline mutation remains below the React workspace in the existing timeline, project, engine, and public command owners. This checkpoint does not create a competing edit model, publish approximate times, add clip mutation controls, or claim thumbnail, waveform, advanced selection, playback, or render ownership assigned to later checkpoints.

## Consumer behavior

Each row exposes stable identity, kind, targeting, synchronization state, and exact record placement. Each item exposes exact source and record ranges plus media, nested-timeline, group, link, transition, and selected-state evidence when present. The ruler and readouts use the timeline rate, the visible range is shaded, the playhead remains legible across every row, and responsive styling keeps controls and track metadata usable at reduced widths.

Malformed format revisions, root mismatches, invalid canonical time, unsafe display magnitudes, missing references, and invalid transition placement or offset clocks fail explicitly. An absent or invalid document produces an honest unavailable surface without changing the last application-owned snapshot or inventing an editable replacement.

## Verification

The focused timeline contract proves strict revision handling, exact range and identity preservation, two-pass transition placement, deterministic ruler and range math, invalid-document rejection, React integration, and the absence of a frontend mutation owner. The adjacent editor-workspace contract, strict TypeScript check, complete frontend suite, production Vite build, refreshed codebase-map validator, and repository checkpoint verifier pass against the real consumer.

## Remaining constraints

Fractional-rate ruler labels are deterministic frame counts because the canonical document does not yet carry a drop-frame or display-timecode mode. Playback observation is presented when available but does not grant transport command ownership. Clip editing gestures, thumbnail and waveform enrichment, advanced selection, synchronized playback, and render evaluation remain separate checkpoints and must reuse this exact canonical projection rather than replacing it with local authored state.
