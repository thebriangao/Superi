# P1.W07.C023

## Outcome

Created a deterministic OTIO 0.18.1 interchange baseline with two timelines and one explicit
expectation record. The baseline preserves clips, gaps, transitions, owner-relative markers,
nested composition, two linear rate changes, stable editorial identity, metadata, media linkage,
and understandable preserve plus diagnose handling for constructs outside the first supported
subset.

## Architecture and integration

`superi-fixture-tool` owns the dependency-free `generate-otio` command. It builds native OTIO JSON
from fixed Rust values, computes a strict schema 1 fixture manifest, and refuses to replace any
existing output path. The `superi-timeline` crate owns the semantic fixture contract without
claiming a production timeline model or OTIO reader and writer before P2.W02.

`canonical-slice.otio` keeps the first continuous editorial slice concrete: one 24 fps V1 track,
one clip trimmed to source `[24, 72)`, a 48-frame sequence, the exact immutable WebM fixture
identity and digest, and the horizontal mirror effect metadata. `interchange-coverage.otio` adds an
adjacent two-clip dissolve, a gap, track and clip markers, a trimmed nested Stack, stable object and
relationship IDs, and 2.0 and 0.5 `LinearTimeWarp.1` effects. The sidecar expectation record pins
OpenTimelineIO 0.18.1 and declares `FreezeFrame.1` plus a generic named `Effect.1` as opaque objects
that require the stable `timeline.otio.unsupported_construct` warning instead of silent loss.

## Implementation process

The synchronized execution base was `a67f3f3`. Generator, CLI, and timeline semantic contracts were
written first. They failed on the absent generator API, unsupported command, and missing canonical
fixture. The implementation then added deterministic OTIO object construction, exact manifests,
no-overwrite behavior, canonical artifacts, reference expectations, contributor documentation, and
future-consumer proof while leaving the production timeline skeleton intact.

Primary semantics were checked against the official OpenTimelineIO timeline structure, time range,
serialized schema, adapter, and schema versioning documentation. The fixture targets
`OTIO_CORE:0.18.1`, where clips are `Clip.2`, markers are `Marker.2`, transition offsets do not alter
track duration, nested Track and Stack objects are composable children, marker ranges use the
owner's time coordinates, and linear time warps change sampled media speed without changing item
duration.

## Verification

Focused generator tests reproduce all four canonical artifacts byte for byte, enforce the two
timeline report and payload bound, and preserve an existing destination. CLI proof covers the exact
success summary, manifest creation, and overwrite failure. The timeline contract proves exact OTIO
schemas, hierarchy, identities, rational source and parent timing, transition relationships,
markers, gaps, nesting, rate scalars, opaque JSON retention, unsupported diagnostics, and canonical
slice linkage.

Fresh external reference validation used official `opentimelineio==0.18.1` on Python 3.12. Both
files loaded through `otio_json`, reported exact durations of 48 and 120 frames at 24 fps, and
remained semantically equivalent after a targeted official write and read. Broader locked fixture,
timeline, workspace, formatting, Clippy, documentation, policy, offline, and map checks are recorded
in the ignored execution log and matching checklist completion note.

## Remaining constraints

This checkpoint establishes trusted interchange evidence, not the production Rust timeline model,
importer, exporter, editor, or graph compiler. P2.W02 still owns those systems and must consume this
baseline through real reader and writer paths, preserve arbitrary unknown schemas and fields as
opaque versioned data, and prove round-trip behavior before runtime interchange can be claimed.
