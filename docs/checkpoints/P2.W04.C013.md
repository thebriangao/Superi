# P2.W04.C013 macOS Audio Unit hosting

## Outcome

Implemented macOS Audio Unit effect hosting through the existing prepared audio graph. The host
preserves exact sample timing, semantic channel order, routing placement, adjacent-block state, and
finite audible output from a source processor through an Audio Unit and the terminal master mix.

## Architecture and integration

`superi-audio::hosting` owns safe component identity, execution policy, configuration, and prepared
processor surfaces. Its private macOS child owns exact AudioToolbox discovery, asynchronous
instantiation, process-location verification, stream and channel-layout negotiation, stable pull
callback storage, planar conversion, rendering, and teardown. Preparation runs on the blocking
background domain, while the successful audio-domain process path uses only preallocated storage
and enters the same single-input `AudioProcessor` boundary as native Rust effects.

Out-of-process loading is required by default and verified from the instantiated component rather
than inferred from the request. Callers may explicitly select audited in-process execution for a
trusted component. Native, callback, malformed-output, and nonfinite-output failures poison the
instance, while complete prevalidation keeps caller output and graph continuity uncommitted.

## Implementation process

The public consumer contract was added first and failed while `hosting.rs` remained a placeholder.
Implementation then added exact FourCC identity, bounded configuration, native lifecycle ownership,
explicit channel semantics, exactly representable sample timestamps, repeated pull-subrange support,
and complete output validation. The real consumer uses Apple's Peak Limiter component
`aufx/lmtr/appl` between a deterministic stereo source and the existing terminal `SummingBus`.

## Verification

Focused callback contracts prove native storage copies, host-plane publication, repeated bounded
pulls, malformed request rejection, and failure reporting. Public macOS contracts prove the real
Apple effect in both audited in-process and verified out-of-process modes, exact adjacent partition
continuity, stereo meaning, finite audible output, and failure atomicity for unrepresentable sample
time. The delivery gates also cover the complete audio package, engine consumers, formatting,
documentation, warnings-denied Clippy, dependency policy, unsafe inventory, codebase maps, and the
full repository checkpoint verifier.

## Delivery

This record ships with the safe host surface, private audited AudioToolbox boundary, real graph

## Remaining constraints

This checkpoint hosts macOS effect Audio Units only. Parameter discovery and automation, presets,
instruments, MIDI, sidechains, plug-in UI, latency compensation, crash restart, scanning, export
adaptation, and VST3 remain separate work. Physical device latency, third-party plug-in coverage,
hardware soak, and the engine adapter from decoded timeline slices through device or export delivery
remain owned by later checkpoints and platform lanes.
