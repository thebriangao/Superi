# P3.W05.C009

## Outcome

Implemented exact monitor color-profile selection and reproducible sRGB and Display P3 transforms
for every native viewer role.

## Architecture and integration

The desktop native owner refreshes the existing bounded display-profile catalog, defaults each
viewer to the primary active monitor, publishes profile identity without ICC bytes, and binds an
explicit role-local selection through `MonitorPresentationBinding`. Every presentation command
carries immutable binding evidence, and the GPU owner checks the latest shared catalog before frame
acquisition and again before submit and present, so a removed monitor or changed profile cannot
silently present through stale color state. A selection change hides the native child until the
first replacement frame presents successfully, and native commands plus shell replies are revision
fenced so rapid choices cannot reveal or publish an older transform.

`ViewerPresentationIntent` now derives either the sRGB or Display P3 branch from the unchanged
scene-linear ACEScg, premultiplied RGBA16F render result. Both paths preserve the existing six-stage
order and build the real `GpuDisplayPresenter` from one explicit `OutputColorTransform`. The React
viewer consumes a frozen catalog projection and sends only role, monitor ID, and transform code
through a separate unknown-field-denying control command; placement remains geometry-only and no
pixels, ICC bytes, matrices, project commands, or playback commands cross the webview boundary.

## Implementation process

Focused frontend and Rust contracts first failed for the missing color-management module,
`ViewerDisplayTransform`, and transform-aware intent constructor. The implementation then added the
smallest pure UI contract, native profile lifecycle, stale binding guard, real presenter selection,
four-role controls, strict IPC coverage, and current map evidence without resetting navigation,
overlays, comparison captures, exact status, or playback state.

## Verification

The focused frontend color, placement-only IPC, and real workspace consumer ring passed 12 tests.
The native library and presentation contracts passed 16 tests, including exact and reversible
monitor selection, stale binding rejection, deterministic sRGB and Display P3 intent, scene
meaning, RGBA16F precision, and transform order. Strict TypeScript, the production Vite build, the
complete 96-test frontend suite, and live macOS system profile discovery also passed before the
repository verifier and final delivery gates.

## Delivery

The profile selection lifecycle, transform-aware native presenter, controls, contracts, maps, and
this record are delivered together. The exact delivered commit is recorded in the matching
checklist completion description after `origin/main` containment is confirmed.

## Remaining constraints

The monitor profile supplies exact identity, model, rendering intent, and freshness evidence. This
checkpoint does not claim arbitrary ICC matrix, TRC, or LUT execution; the selected executable
display branch is the explicit built-in sRGB or Display P3 transform, and targets without native
system profile discovery remain visibly unavailable rather than receiving a fabricated profile.
