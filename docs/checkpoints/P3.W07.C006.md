# P3.W07.C006

## Outcome

Implemented keyboard-only movement across the always-mounted route, workspace-control, and active
workflow regions. F6 and Shift+F6 enter, advance, reverse, and wrap predictably, while a first-focus
skip link moves directly into the active workflow.

## Architecture and integration

`app/src/focus-management.ts` extends the existing shared focus boundary with a pure F6 input
classifier, deterministic landmark indexing, operable landmark discovery, and no-scroll focus
movement. `ApplicationShell` registers the three real regions and ignores modal descendants so
palette and menu focus containment remains authoritative. `PanelWorkspace` exposes the current
route's mounted panel layout as the stable skip target without changing reducer or project state.

## Implementation process

Focused contracts first established the missing region-level traversal. The implementation reused
the delivered focus target policy, added no new dependency or shortcut persistence, and preserved
native tab order, tablist arrow navigation, editable controls, composition, repeated input, and
modal focus traps.

## Verification

Focused Node contracts cover F6 eligibility, modifiers, composition, repeat handling, outside
entry, bidirectional wrap, all three real shell landmarks, modal exclusion, the skip target, and
theme-owned visible focus. Strict TypeScript passes, the complete frontend suite passes 196 tests,
the production build transforms 104 modules, and all 26 module maps plus the global index validate
from the final synchronized tree.

## Delivery

remain platform certification evidence. This checkpoint does not alter authored project state,
