# P1.W07.C026

## Outcome

Made canonical fixture validation and the complete eight-stage editorial slice contract mandatory
across every hosted Rust build job. The shared baseline now uses a portable version 2 expectation
record, so identical project state has one digest across clones and isolated worktrees.

## Architecture and integration

The cross-platform workflow directly runs the locked fixture validator and normalized
`superi.slice.canonical.v1` command after the workspace quality suite in both declared build jobs.
Each job also compiles and tests the supported `os-codecs` CLI configuration, whose report now
records the active feature instead of incorrectly identifying every build as default-only.
The platform matrix names this portable evidence `slice-contract` and retains `slice` for physical
all-runtime proof, so hosted success cannot stand in for GPU, display, audio, hardware codec,
rendered-pixel, playable-export, performance, or soak evidence.

The canonical runner still derives its stub inventory from the eight stage records. Contributor
guidance requires a production subsystem to route its matching stage through the real owner,
preserve the stable stage ID and instrumentation probes, replace the stub diagnostics, and add
consumer proof in the same change. Compilation, a capability label, or a metadata-only readiness
flag cannot make a stage runtime.

Expectation version 1 remains immutable. Version 2 preserves the same source, audio parents,
reference frames, timestamps, tolerances, and export intent, but replaces the absolute authoring
worktree path with the stable repository-relative fixture identity before hashing project state.
Undo and redo comparison still retains the observed path, while only the portable expectation
digest removes checkout location from semantic identity.

## Implementation process

The workflow contract was added first and failed because two declared build jobs contained zero
explicit fixture or slice commands. Adding both commands to both jobs made that focused contract
pass.

The first direct baseline run then exposed a second red condition: the C024 project-state digest
captured its authoring worktree path, so the real normalized command failed in another worktree.
A focused normalization contract, immutable version 2 fixture, strict resolver update, and improved
digest diagnostic corrected the actual consumer rather than weakening expectation verification.

## Verification

The focused workflow contract proves one exact fixture command and one exact normalized slice
command plus locked default and `os-codecs` configuration coverage per declared build job. The
portable-state unit contract proves two checkout roots produce the same canonical project digest,
and the process contract executes the real command twice with version 2 expectations, exact report
comparison, and accurate active-feature identity.

The fixture validator reports ten valid versions and 23 payloads. A fresh direct command reports
contract success, project digest
`15628621f9e49cdab04ff1623474f7cc4ea6f175a38d7b6ab95722b84403a63b`, all eight stages, passed
version 2 expectation evidence, and the six unchanged production gaps. Broader workspace, policy,
formatting, strict lint, documentation, dependency, workflow, and map results are recorded in the
ignored execution log.

## Delivery

This record is committed with the hosted baseline, portable expectation version, consumer tests,
contributor and platform guidance, and current codebase maps. The delivery commit containing this
record is pushed to `origin/main`; its SHA is recorded in the matching Google Docs completion
description and ignored execution log after it exists.

## Remaining constraints

The hosted `slice-contract` lane is continuously working contract evidence, not a playable editor
export or all-runtime conformance. Six stages remain honest stubs until their scheduled production
subsystems exist and integrate through the real public API consumer. Physical hardware results and
long-running performance evidence remain owned by the named platform lanes.
