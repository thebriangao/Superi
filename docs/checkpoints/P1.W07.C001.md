# P1.W07.C001

## Outcome

Created GitHub Actions jobs that build the locked open workspace on documented macOS, Windows, and
Linux hosted runners. Pull requests and `main` exercise the current blocking lanes plus the Ubuntu
26.04 preview lane, while Ubuntu 22.04 runs on the documented weekly or manual cadence.

## Architecture and integration

The workflow lives at `.github/workflows/ci.yml`, the repository-owned CI boundary. It maps stable
lane IDs from `docs/platform-testing.md` to explicit GitHub-hosted runner labels, keeps hosted build
evidence distinct from physical GPU, display, audio, and hardware-codec evidence, and builds every
member of the `open/` Cargo workspace through its locked dependency graph.

The workflow grants only read access to repository contents, disables persisted checkout
credentials, pins `actions/checkout` to an immutable commit, cancels superseded branch runs, and
keeps matrix fail-fast disabled so every platform reports independently. Ubuntu 26.04 is the only
nonblocking lane because GitHub still classifies that image as public preview.

## Implementation process

The final verified base was `e0b3af9f099f527a8544d1b0317896640969903b`, rebased from the initial
synchronized base after the dependency-policy workflow and its executable contract landed.
Repository and workstream mapping found no earlier cross-platform CI workflow, then current official
runner and workflow-security documentation was checked before the workflow was introduced. A
job-level cadence design was corrected during review because matrix values are unavailable when
GitHub evaluates a job-level condition, so the Ubuntu 22.04 cadence is represented by a separate
scheduled or manual job.

## Verification

A focused contract first failed because `.github/workflows/ci.yml` did not exist, then passed YAML
parsing and exact assertions for triggers, permissions, concurrency, six lane-to-runner mappings,
preview policy, immutable checkout, disabled persisted credentials, locked builds, and neighboring
checkpoint scope. `cargo build --workspace --locked` passed with stable Rust 1.97.0 using the locked
dependency set, and all seven `superi-fixture-tool` policy tests plus its documentation tests passed
offline. Diff hygiene and the repository prose-dash prohibition were also checked.
