# P3.W02.C002

## Outcome

The native editing viewport now has an explicit control-only Tauri command boundary. Its placement
payload rejects unknown fields, so frame bytes, encoded images, pixel payloads, blob references,
and GPU texture handles cannot be accepted by the native presentation command.

## Architecture and integration

The delivered C001 presenter remains unchanged: Tauri owns the native child window and the managed
GPU submission thread acquires, renders, submits, and presents the surface texture directly. React
may send only `x`, `y`, `width`, `height`, `scaleFactor`, and `visible` through
`desktop_viewport_update`, while the returned snapshot contains status metadata rather than media.
Serde unknown-field denial closes the command payload without adding a second presenter, encoded
fallback, public media route, or viewer-specific behavior.

## Implementation process

The checkpoint synchronized to the delivered C001 commit
`1940765a5d9b3e887b5dfd3791c20b303cebe4e6` and preserved its direct GPU path. The focused contract
was frozen around the existing command seam, its two mismatched source assumptions were corrected
from red output, and the production change was limited to strict deserialization of the existing
placement DTO.

## Verification

`node --test app/tests/native-viewport-ipc-contract.test.mjs` passed the control-only invocation and
no-webview-image-path contract. With
`CARGO_TARGET_DIR=/private/tmp/superi-targets/P3.W02.C002`, the locked desktop viewport test slice
compiled and passed both unit tests with no failures.

## Delivery

The strict placement guard, focused contract, checkpoint record, and semantic map updates ship
together in one Conventional Commit on `origin/main`. The live checklist completion records the
exact remotely contained commit after push.

## Remaining constraints

This checkpoint does not add C003 viewer roles, viewer source selection, or alternate presentation
modes. Physical display, GPU adapter, color-profile, and resolution coverage remains the platform
evidence responsibility of the native presentation path rather than an IPC fallback.
