# P3.W07.C015

## Outcome

Implemented deterministic frontend path construction, explicit case-sensitive or insensitive
comparison keys, exact Unicode filename preservation, and removable-volume behavior projection.

## Architecture and integration

The pure filesystem path owner normalizes separator syntax and NFC only for explicit comparison
keys, never for I/O paths. Project naming and batch relink preserve original filename code points,
while the media inspector projects the existing authoritative native mounted, offline, missing,
changed, and unavailable source evidence into one stable action state.

## Implementation process

A failing contract established POSIX and Windows separators, composed and decomposed Unicode,
explicit case policy, removable-volume transitions, and the real App consumer before integration.

## Verification

Thirty-nine focused filesystem, application, desktop shell, and project history tests, 222 frontend
tests, strict TypeScript, the 108-module production build, and codebase-map validation pass.

## Delivery

Path policy, real consumers, tests, evidence, and maps ship together.

## Remaining constraints

Filesystem case sensitivity is deliberately caller-supplied rather than guessed from platform;
native source monitoring remains authoritative for physical volume and byte availability.
