## Outcome

Implemented the fixed headless runner for `superi.slice.canonical.v1` through the public API. The
runner executes exact canonical editorial state, proves reversal, emits all eight required stage

## Architecture and integration

`superi-engine::command` owns bounded fixture identity, the canonical one-track edit, the complete
three-node mirror graph, four stable mutation records, and full-state undo plus redo. `superi-api`
projects that state through one strict versioned typed command. `superi-cli` validates repository
fixture authority, executes the normalized fixed invocation, writes a collision-safe strict report,
and publishes `canonical.webm.contract-stub` because the production import, timeline, graph, color,
and muxing owners are not integrated.

## Implementation process

Focused engine, API, and process contracts were revised first from a provisional generic runner to
the delivered normative slice. The red contracts exposed missing fixture identity, exact timeline
and graph state, operation evidence, fixed argument parsing, stage records, output safety, and
conformance disclosure. The implementation then replaced the generic action surface, removed export
from the mutation history, added the required encoded fixture dependency, and coupled the real
network-isolated consumer to the canonical command.
