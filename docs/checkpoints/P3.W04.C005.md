# P3.W04.C005 exact timeline snapping

## Outcome

The production timeline canvas now snaps playhead and range gestures to exact canonical timeline, playhead, item-start, item-end, marker-start, and marker-end targets. A visible guide and live exact-time consequence identify every accepted target, while a session master switch and six discoverable target rules make the active behavior explicit without implying authored persistence.

## Architecture and integration

`superi-timeline` remains the snapping-policy and authored-state owner. The existing strict `projectTimelineDocument` consumer now projects immutable target identities from the canonical document, retains every eligible target as an exact point in the timeline edit clock, and exposes a pure `resolveTimelineSnap` query for transient canvas gestures. The React workspace does not call an edit command, publish a second timeline model, or turn session presentation settings into project state.

Timeline start and direct timed-item edges resolve from exact record ranges. Timeline and track markers resolve in their owner's record clock. Object markers resolve relative to their owner's exact record start, while valid markers beyond the object duration remain unavailable. Any candidate that cannot rescale exactly into the timeline edit clock is skipped, matching the lower exact query instead of rounding an audio or alternate-clock coordinate into a false frame target.

The resolver first compares exact edit-frame distance, then the stable lower target order: timeline start, playhead, item start, item end, marker start, and marker end. Stable target identity and exact coordinate finish tie resolution. The canonical persistent snapping switch remains authoritative, and a disabled project or session switch produces no target match while pointer and keyboard timing stay frame precise.

## Editorial gestures and visible state

Pointer coordinates are frame aligned before target resolution. Playhead drags exclude the moving playhead itself, while in-point and out-point drags may snap to the current playhead. A small pixel tolerance converts to an integer edit-frame window and is capped across wide views, so zoom does not create unbounded magnetic behavior.

The session strip exposes one master control plus independent timeline-start, playhead, item-start, item-end, marker-start, and marker-end rules. During a match, the canvas draws a high-contrast vertical target guide and announces the moving tool, target label, exact resulting time, and distance. An unsnapped gesture reports that it remains frame aligned. Keyboard nudges remain direct frame steps and clear stale target feedback.

Every pointer gesture captures its original transient coordinate. Escape restores that coordinate, clears the target, releases pointer capture, and ends the gesture immediately. Pointer cancellation follows the same reversal path, while pointer release accepts the visible result and retains the last matched guide until a new unsnapped action or rule change.

## Failure handling

Duplicate marker identities, absent owners, unsupported owner kinds, invalid owner clocks, negative marker starts, malformed ranges, and invalid track record clocks fail the strict projection visibly. Valid object-marker overscan and exact-rescale failures omit only the unavailable candidates. Existing invalid-document handling keeps the canvas honest rather than fabricating target state.

## Verification

The focused timeline contract proves timeline, track, and object marker resolution, object-relative offsets, inexact audio-clock omission, overscan omission, deep immutability, stable tie ordering, per-kind filtering, canonical and session disablement, malformed-owner failure, production canvas wiring, visible state, and Escape reversal wiring. The complete frontend suite, strict TypeScript check, Vite production build, lower marker contract, updated codebase maps, and repository checkpoint verifier exercise the real `EditorWorkspaces` consumer and lower snapping owner.

## Remaining constraints

The session rule strip is intentionally transient until a public authored-settings command exists. This checkpoint applies snapping to the current playhead and range gestures; later authored clip-edit tools must call the lower timeline operation and snapping owners through the public command boundary, reuse the exact target contract, expose their own edit consequence, and preserve immediate reversal through command history.
