# P3.W05.C011

## Outcome

Implemented frame and audio cache indicators that preserve exact viewer state without adding a blocking interaction, command, timer, polling loop, or second state owner.

## Architecture and integration

The existing frozen role-aware viewer status projection now adds `Frame cache` beside playback scheduling and `Audio cache` beside transport audio state. Frame cache presentation retains the exact transport mode, foreground scheduled frame, due clock, predictive failure, and decoded-output availability, explicitly labels fill, hit, occupancy, and prediction completion as unavailable when the public snapshot does not expose them, and states that interaction does not wait for cache work.

Audio cache presentation preserves callback discard generations and synchronization state, then projects each canonical audio track in stable order with its exact sample rate, ordered source channels, destination and ordered destination channels, complete channel or mute routing, clip count, and audited or unsupported continuity seams. Invalid timing, identity, channel, route, or continuity evidence fails closed; production `audio_output_unavailable` remains explicit, so the indicator never infers samples, signal level, device output, or audible continuity that no runtime owner observed.

Both indicators remain synchronous immutable strings derived from the existing `EditorStateSnapshot`. `NativeViewport` continues to render the single field catalog outside its transformed frame and native placement host, so paused, playing, scrubbing, ended, fit, zoom, pan, pixel, fullscreen, cinema, comparison, analysis, overlay, and external-display behavior retain the same exact temporal, visual, audio, and comparison owners.

## Implementation process

Focused contracts first failed because the two fields and catalog entries did not exist. The implementation then extended only `ViewerStatusDisplay`, its field catalog, unavailable projection, and pure formatter helpers, followed by the real workspace mounting contract and durable mapping evidence.

Strict typecheck, production build, the complete frontend suite, map validation, and the repository checkpoint verifier are recorded in the checkpoint execution evidence before delivery.

## Remaining constraints

The public playback snapshot does not expose frame-cache occupancy, hit rate, byte count, predictive completion, or audio-buffer fill. The production desktop runtime is timing-only and reports unavailable decoded viewport and device audio output, so these indicators communicate exact scheduling, degradation, acknowledgement, and authored audio meaning without claiming decoded frames or audible samples.
