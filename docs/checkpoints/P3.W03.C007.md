# P3.W03.C007

## Outcome

The project media inspector now exposes exact fingerprint-based duplicate identity and persists reusable source-time selections with manually refinable tracked regions as ordinary editable media state.

## Architecture and integration

The C006 `MediaBrowserItem` remains authoritative for source facts, generic user metadata, typed annotations, and derived usage. C007 adds a nonserialized `MediaIdentityTracking` projection derived only from stable media IDs and authoritative content fingerprints, plus a separate persisted `selections` collection.

Each selection uses an exact integer frame interval and rational rate. Stable tracked-region identities retain ordered manual observations with fixed-point normalized geometry, avoiding floating-point persistence and keeping temporal refinement deterministic.

One revision-fenced Tauri command atomically replaces the bounded selection collection for a stable media ID. Candidate validation rejects stale revisions, duplicate identities or times, invalid ranges, out-of-bounds geometry, and oversized state before publication while preserving every C006 field.

## Implementation process

The smallest implementation then extended the existing native media-library snapshot, strict TypeScript bridge, registered Tauri command, and selected-media React detail without adding proxy, optimized-media, relink, search, perceptual matching, or automated tracking behavior.
