# P3.W04.C004 timeline selection and navigation

## Outcome

The production timeline canvas now supports deterministic click, related, direct, range, lasso,
and keyboard selection through the existing shared application selection owner. Group members
select as one editorial unit, linked clips join that fixed-point expansion only when canonical
linked selection is enabled, and Option preserves exact-object control.

## Architecture and integration

`TimelineEditState` remains the sole authored owner of selected objects, clip groups, clip links,
and the linked-selection preference. The React canvas reads those canonical relationships from the
strict timeline projection and never creates a project command, another editable timeline model, or
an authored group or link mutation path.

The interaction selection is stored as exact `superi.editor.state` references in the existing
immutable `ApplicationState.selection`. Each reference carries the public schema, current project
revision, root timeline identity, item kind, and stable item identity. Foreign, malformed, stale,
or missing references do not become selected canvas objects.

Pure timeline helpers own stable identity encoding, topmost-first selection order, fixed-point
relationship expansion, contiguous range resolution, nearest temporal vertical navigation,
same-track horizontal navigation, and normalized rectangle intersection. Lasso uses the actual
mounted option rectangles for hit testing, while range and keyboard selection use the complete
projected model and therefore remain deterministic beyond the mounted overscan window.

## Interaction behavior

Ordinary click replaces selection with the selected item's group and enabled link closure.
Command or Control-click toggles that closure, Shift-click selects a contiguous display-order range,
and Option-click bypasses relationships for exact-object control. Dragging empty track space moves
the playhead and previews one lasso selection; pointer release commits once, while pointer
cancellation discards the preview and restores the gesture's original playhead.

Each item participates in one roving keyboard focus order. Left and right move through one track,
up and down choose the nearest item on the adjacent populated track, Home and End reach track
boundaries, Shift extends from the shared anchor, Command or Control-A selects all, Enter or Space
selects the focused object, and Escape clears. Navigation scrolls offscreen targets into the
virtualized window before focus moves.

The canvas exposes a labeled multiselect listbox, options with exact `aria-selected` state, a polite
live selection count, visible focus, pointer and keyboard instructions, responsive wrapping, and
distinct authored-selection evidence. Existing ruler, playhead, range, scroll, zoom, fit, strict
projection failure, and topmost-first track behavior remain intact.

## Remaining constraints

This checkpoint consumes existing canonical groups, links, authored selection, and linked-selection
preference. Creating, dissolving, persisting, undoing, or exposing those authored operations through
the public project command surface remains with their lower timeline, project, engine, and API
owners. Later timeline edit checkpoints must reuse the shared application selection references and
must not replace them with component-local authored state.
