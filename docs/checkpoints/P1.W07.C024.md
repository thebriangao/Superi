# P1.W07.C024

## Outcome

Created the revisioned expected contract for the canonical editorial slice. The fixture records
exact frame hashes, pixel tolerance, synchronized audio samples, timestamps, project states, and
export metadata so contributors and CI can reproduce and verify the first slice offline.

## Architecture and integration

`slice/expectations/v1` is a derived fixture with exact parent references to the canonical AV1
slice source and synchronized multichannel audio baseline. Its RGBA8 payload contains 48 mirrored
reference frames, and its strict JSON record binds whole-payload and per-frame SHA-256 identities,
timestamps, state digests, delivery traits, and explicit comparison tolerances.

The private `superi-cli` expectation resolver is the real consumer. It performs bounded,
non-symlink reads, validates strict manifest and record schemas, verifies both parent manifests,
hashes every reference frame, inspects WAVEFORMATEXTENSIBLE timing and channel meaning, checks
sample probes and continuity, and compares the modeled state, timestamp, and export observations.
Its report distinguishes applicable contract success from rendered pixel work that remains
unevaluated and from rendered audio that is not applicable to the video-only slice.

## Implementation process

The synchronized base was `218e110`. The scenario contract was tightened first and failed because
the report still returned `unavailable`. The fixture, resolver, observation wiring, failure
classification, and stable evidence were then added until the same real CLI workflow passed.

Reference frames were derived from source frames 24 through 71 with FFmpeg 8.1.1 using the exact
trim, horizontal mirror, timestamp reset, and RGBA conversion recorded in the manifest. Audio
expectations use the existing deterministic PCM16 parents and preserve exact sample clocks,
ordered speaker layouts, common onset and tail timing, routing signatures, and the verified
adjacent-sample continuity bound.

## Verification

Focused unit tests prove the canonical record, reject a corrupted reference payload before
comparison, and classify state drift as a contract mismatch. The real scenario test executes the
normalized public command twice, proves stable expectation evidence, and verifies the exact frame,
audio, timing, state, and export contract report.

The generic fixture validator proves schema, provenance, lineage, inventory, sizes, hashes, and
safe paths for the derived version. Broader CLI, workspace, policy, formatting, documentation, and
map results are recorded in the ignored execution log.

## Delivery

This record is committed with the canonical expectations, strict CLI consumer, contributor
documentation, tests, and current codebase maps. The delivery commit containing this record is
pushed to `origin/main`; its SHA is recorded in the matching Google Docs completion description and
ignored execution log after it exists.

## Remaining constraints

The checkpoint provides trustworthy expected data and verifies every contract observation that the
current slice actually produces. It does not turn the disclosed graph, color, or export stubs into
runtime implementations, compare rendered pixels that do not yet exist, claim a playable export,
or add audio to the video-only canonical source. Later runtime owners can consume this stable
record without changing its versioned meaning.
