# P3.W07.C001

## Outcome

Implemented one shared shell-input contract for deterministic keyboard routing, stable focus
restoration, safe pointer capture, precision pen samples, and fractional high-resolution wheel
input. The production application, dock separators, and timeline reuse it without moving authored
project, audio, timing, routing, history, or persistence authority into React.

## Architecture and integration

`app/src/shell-input.ts` is a pure bounded normalization layer. It classifies handled, composing,
repeated, and editable keyboard input; restores only connected operable focus targets without
scroll jumps; retains coalesced mouse, touch, and pen identity plus pressure, tilt, twist, contact,
and timestamp evidence; contains pointer-capture failure; and converts pixel, line, and page wheel
deltas to finite fractional pixels.

`ApplicationProvider` applies the shared eligibility rule after resolving an exact registry-backed
shortcut. `PanelWorkspace` restores tab focus through the safe helper and releases dock-resize
capture on completion or cancellation. The existing timeline keeps native unmodified scrolling,
uses normalized shifted pan and pointer-anchored zoom, and consumes the latest coalesced coordinate
for playhead, range, lasso, and edit gestures.

## Implementation process

The focused contract first failed because `shell-input.ts` did not exist. Implementation then added
the pure model and wired only the existing owners. A test-only strict-reference assertion was
corrected to structural equality after confirming the helper intentionally returns a fresh deeply
frozen sample on each normalization call.

## Verification

Focused shell-input, keyboard-shortcut, application-framework, and timeline contracts pass 32
tests. They cover IME and editable routing, focus fallback, coalesced pen metadata, pointer capture
release, fractional pixel, line, and page wheel conversion, application routing, panel layout, and
the real timeline consumer. The complete reconciled frontend suite passes 172 tests, strict
TypeScript passes, the production build transforms 96 modules, and all 26 module maps plus the
global index validate.

## Delivery

The shared contract, three production integrations, focused tests, package gate, checkpoint record,
and refreshed maps are delivered together by the commit containing this file. The matching live
checklist records the exact commit only after origin/main containment is confirmed.

## Remaining constraints

Browser Pointer Events and Wheel Events remain the platform abstraction, so native tablet driver
configuration and physical device certification stay outside this checkpoint. Default unmodified
scrolling remains browser-native, no input samples are persisted, and audio timing, channels,
routing, synchronization, and audible continuity remain owned and observed by the existing lower
systems.
