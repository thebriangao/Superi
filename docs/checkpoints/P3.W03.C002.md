# P3.W03.C002 Project settings

## Outcome

Superi projects now persist and expose editable frame rate, resolution, color, audio, cache,
proxy, and working-folder settings through the delivered desktop project lifecycle. Audio sample
rate and output channel layout remain exact project authority, without adding alternate timing,
routing, synchronization, or continuity state.

## Architecture and integration

`superi-project` remains the sole durable settings owner and now accepts three optional bounded
working-folder keys for general work, cache, and proxy locations. The desktop lifecycle projects
the existing public settings snapshot into one strict presentation DTO, executes one optimistic
atomic transaction through `LocalProjectHost`, and refreshes the active lifecycle identity only
after durable success. Two shell-local Tauri commands and their typed React wrappers let the System
panel inspect and update the same project authority.

## Implementation process

Implementation was dependency-gated on P3.W03.C001 commit
`1121145c00a626a4be1f58548ce54612b0b9372f`. A frozen integration contract was added first and
observed red for the missing desktop DTO and lifecycle methods. The minimum durable vocabulary,
lifecycle bridge, command registration, typed client, and compact settings editor then made that
contract green without changing lifecycle flows or adding C003 media-organization behavior.
