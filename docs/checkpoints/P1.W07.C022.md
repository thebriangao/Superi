# P1.W07.C022

## Outcome

Created a deterministic media error baseline covering malformed, truncated, unsupported, and
partially readable PCM containers. The versioned baseline contains four tiny media cases, one strict
catalog, exact provenance, and reproducible byte identities for contributor and CI use.

## Architecture and integration

`superi-fixture-tool` owns the dependency-free generator and `generate-media-errors` command. It
emits fixed WAVE and AIFF bytes, applies controlled block-alignment, truncation, and form-type
changes, serializes a 14-field CRLF catalog, creates the schema 1 manifest, and refuses every
existing output path. `superi-media-io` consumes the checked-in cases through the production
`PcmContainerSource` parser and packet reader without adding a runtime dependency on the tool.

The partial case follows the production source lifecycle instead of weakening strict parsing. Its
canonical WAVE seed is complete at open, the consumer applies the cataloged truncation to a
temporary path after open, and `read_packet` returns the aligned usable prefix only as
`ReadOutcome::Partial` with exact corruption evidence. The other cases prove malformed WAVE and
truncated AIFF `corrupt_data` errors plus the explicit `unsupported` AIFC boundary.

## Implementation process

The synchronized base was `a67f3f32773ad895b3e783c60d114f71d27e8294`. Generator, CLI, and real
consumer contracts were written and
run first. The generator contract failed because the media-error API did not exist, the CLI
contract failed because `generate-media-errors` was unsupported, and the media-I/O contract failed
because the canonical catalog and payloads were absent. The implementation then added deterministic
serialization, the command surface, canonical artifacts, strict catalog parsing, byte-level
mutation checks, and production-interface proof.

The repository's current parser and shared error vocabulary determined the cases and expected
outcomes. In particular, strict open rejects a source whose declared bytes are already absent, so
the only honest partial-read fixture is a complete seed paired with an explicit post-open
truncation recipe. No network source, external media tool, random value, or platform codec
participates in generation or proof.

## Verification

Focused generator tests prove two independent outputs match one another and every canonical file
byte for byte, with exact case counts, tiny size bounds, and preserved existing destinations. CLI
tests prove the success report, manifest creation, complete usage, and overwrite refusal. The
media-I/O contract independently verifies the fixed catalog, header mutations, declared truncation,
production open classifications, aligned partial packet, and exact corruption kind, recovery,
stream, byte offset, expected byte count, actual byte count, usable byte count, and frame count.

The canonical fixture validator covers provenance, inventory, byte count, digest, safe paths, and
malformed manifest behavior. Broader fixture-tool, media-I/O, workspace, policy, formatting, and map
validation results are recorded in the ignored execution log.

## Delivery

This record is committed with the generator, CLI, canonical artifacts, consumer contract, fixture
documentation, and current codebase maps. The delivery commit containing this record is pushed to
`origin/main`; its SHA is recorded in the matching Google Docs completion description and ignored
execution log after it exists.

## Remaining constraints

These fixtures prove the current WAVE, AIFF, and AIFC parser and short-read boundary. They do not
claim malformed coverage for every implemented container or codec, hardware decoder behavior,
playback recovery, fuzzing completeness, UI diagnostics, or the complete editorial slice. Later
checkpoints may expand the suite while retaining this versioned identity and its exact outcomes.
