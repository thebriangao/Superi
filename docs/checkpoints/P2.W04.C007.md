# P2.W04.C007 Common channel layouts

## Outcome

Superi audio now exposes canonical mono, stereo, quad, six-channel surround, and eight-channel
surround layouts plus explicit speaker and discrete channel conversion. Speaker conversion uses
documented coefficients for mono through six-channel surround, while undefined eight-channel
speaker conversions fail closed.

## Architecture and integration

`superi-audio::channels` builds on the ordered semantic `ChannelLayout` owned by `superi-core` and
implements the existing audio graph `AudioProcessor` contract. A `PreparedChannelMixer` computes
its immutable matrix before callback processing, validates exact runtime layouts and sample counts,
and writes interleaved output without allocation, locks, timing changes, or implicit graph routes.
Direct graph edges still require exact layout equality, so every conversion remains an explicit
node selected by the caller.

## Implementation process

The public integration contract was added first and failed because the channel module did not
exist. Implementation then added the common-layout catalog, speaker and discrete interpretations,
precomputed coefficient matrices, fail-closed validation, and crate-root export. The focused test
also inserts the converter into a prepared graph and processes consecutive audio-domain blocks.

## Verification

Focused tests prove canonical channel order, common-layout recognition, documented six-channel
downmix coefficients, discrete drop and zero-fill behavior, undefined conversion rejection,
unchanged output on validation failure, and exact consecutive sample time through the prepared
graph. The complete audio package, synchronized multichannel media consumer, formatting,
warnings-denied Clippy, documentation, map validation, and changed-path checkpoint verification are
delivery gates.

## Delivery

The delivered commit is recorded in the live Phase 2 descriptions tab after remote containment is
confirmed. The matching main-tab specification remains unhighlighted until fresh verification on
the delivered tree passes.

## Remaining constraints

No universal normative eight-channel speaker downmix was identified, so only exact speaker-layout
identity is accepted for that layout and callers may explicitly choose discrete index conversion.
Physical speaker labels and device-layout negotiation remain platform-adapter and hardware-lane
responsibilities; this checkpoint does not implement effects, metering, plugins, automation, or
sample-rate conversion.
