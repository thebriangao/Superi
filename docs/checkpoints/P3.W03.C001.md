# P3.W03.C001

## Outcome

Implemented production desktop create, open, close, save, save-as, recent-project, and recovery
flows. Failed operations retain the last valid active and recent state together with a stable code,
title, action, reviewed context, and exactly one retryable, degraded, user-correctable, or terminal
classification.

## Architecture and integration

One serialized Tauri project lifecycle owns only desktop session presentation. Its concrete backend
delegates durable work to `superi-api::LocalProjectHost`, which now exposes collaborative-safe save
and collision-aware save-as alongside existing create, validation, and revision-fenced recovery.
Active identity and bounded deduplicated recents change only after the local host proves success.

The Tauri state is initialized with the application recovery root and exposes one snapshot command
plus one complete command executor. A strict TypeScript adapter and the existing System panel form
the real desktop consumer without moving project persistence, settings, media organization,
relinking, proxy generation, or search into React.

## Implementation process

The frozen lifecycle contract first failed because the desktop project lifecycle module did not
exist. The smallest state owner, local-host adapter, Tauri registration, and React consumer then made
the same contract pass while preserving opaque recovery candidate identity, catalog revision
fences, commit-only state transitions, deterministic recent ordering, and actionable failures.

The work synchronized first at `6ddba71dc0d9d62d062185731f5e38a9f1a8bf1e` and later preserved the
non-overlapping viewer delivery at `2faf79bbb00dceca5bd5ab4cccbfe21e50f9d12e` before final map and
delivery records were written.

## Verification

The focused Rust lifecycle contract passes one complete create, save, save-as, open, recent,
recovery, four-failure-class, and close sequence. Strict TypeScript checking and the production Vite
build pass for the typed Tauri adapter and System-panel consumer, and Rust formatting completes on
the changed source and contract.

The affected desktop package compiles all targets, but its unchanged `transport.rs` unit test still
rejects an existing noncanonical public schema context; targeted warnings-denied Clippy likewise
stops only on existing large `PublicApiError` results in that unchanged transport surface. These
out-of-scope baseline failures were preserved rather than absorbed into this checkpoint. The final
source-backed codebase map validation covers the delivered API and workspace maps.
