# P3.W01.C006

## Outcome

Implemented editing, compositing, color, audio, and delivery workspaces as five registry-backed
views over one public editor snapshot owned by the existing application provider.

## Architecture and integration

The delivered `ApplicationProvider` remains the sole application and project presentation owner. It
requests `superi.editor.state.get` through C004's unchanged generated client, rejects stale
completions, retains the last valid snapshot and classified public failure, and refreshes from
generated project, audio, and job events. The five panels read that same snapshot and C005 shared
selection; no second provider, router, transport, retry loop, project store, or editor semantic path
was introduced.

Audio presentation copies the exact public track state and preserves sample rate, source and
destination channel order, route targets including mute intent, playback audio observation, record
gap or overlap sample counts, source discontinuities, and explicit unsupported continuity reasons.
Editing, compositing, color, and delivery likewise display only generated snapshot fields and retain
attached, detached, empty, loading, degraded, and unavailable distinctions.

## Implementation process

Implementation began from synchronized dependency commit
`9f2fc82f1586f2fb800f2d08711b86cbff47762b`, which contains C001 through C005. The focused C006
contract first failed because the editor projection module did not exist. The implementation then
added pure projections, extended the existing provider with one presentation cache, registered the
five workspace views, and added responsive state and audio evidence surfaces without changing C004
transport semantics or C005 application reducers.
