# P3.W03.C011 editable content search

## Outcome

The desktop media library now performs revision-fenced local content search across metadata, transcript artifacts, and local AI content entries, returning deterministic rankings and concise signal-specific evidence. Language analysis is persisted as ordinary editable project-media state with exact frame timing, rational rate, optional speaker identity, and stable timeline plus clip relationships. Saved artifacts remain inspectable and searchable with the source offline and without a model runtime.

## Architecture and integration

The existing Tauri project-media store remains the sole authority. `MediaContentAnalysis` is a backward-compatible defaulted field beside annotations, selections, and derived representations. It binds analysis to one exact source fingerprint, retains supplementary provenance, bounds all collections and encoded size, canonicalizes stable identifiers, and validates every segment link before atomic replacement. Relink and proxy changes preserve the analysis; intentional source replacement retains it as visibly stale until an editor confirms it against the new source.

`MediaContentSearchRequest` fences the active project and library revisions. The native owner searches imported, source, user, editorial, availability, transcript, speaker, relationship, and local AI fields, requires complete token coverage, applies fixed integer weights, caps evidence and result counts, and tie-breaks by stable media identity. Each transcript or linked local-content match can carry exact source frames, rate, speaker, and timeline relationships back to the UI.

The typed TypeScript bridge exposes complete replacement and read-only query commands. The production React media browser debounces current-revision searches, rejects late results, preserves bin and smart-collection scope, presents the strongest explainable match, and provides structured editing for provenance, transcript segments, timing, speakers, timeline relationships, local labels, terms, and segment links. No network client, remote fallback, model dependency, or hidden inference state was introduced.

## Implementation process

The focused contract was added first and failed because the content-analysis owner did not exist. The native artifact, validation, persistence, mutation, and query slice was then completed before the TypeScript bridge and production consumer were wired. Existing media identity, annotation, derived-media, and offline contracts were retained as compatibility gates.

It provides the ordinary artifact and query boundary that an audited later local inference owner can populate. Preview generation, batch operations, changed-volume monitoring, and source-monitor editing remain with their adjacent checkpoints.
