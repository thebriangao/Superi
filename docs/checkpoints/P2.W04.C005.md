# P2.W04.C005

## Outcome

Implemented typed submix, auxiliary, send, return, and single-master routing through the independent
audio graph. Prepared buses sum exact-layout interleaved samples in stable route-identity order and
preserve exact consecutive sample windows through final master output.

## Architecture and integration

`superi-audio::graph` now distinguishes source, ordinary processor, submix, auxiliary, and master
nodes plus direct, send, and auxiliary-return edges. Graph editing rejects duplicate masters,
master outputs, illegal route endpoints, cycles, ambiguous ordinary inputs, and unequal ordered
channel layouts before mutation. Existing source and single-input processor constructors retain
their prior behavior.

Preparation resolves every input route to an earlier runtime node in ascending edge identity order
and retains those bindings beside preallocated node buffers. `AudioProcessInputs` creates borrowed
views over current-block samples without copying, allocation, locking, or self-referential storage.
`prepare_master` selects the one authored terminal master and otherwise uses the same destination
preparation and exact continuity path.

`superi-audio::routing::SummingBus` clears its prepared output and adds each exact-layout input in
the immutable prepared order. The implementation is unity-only: clip gain, fades, pan, mute, solo,
phase, channel mapping, effects, and automation remain with their later owners. Timeline schedule
binding, decoded sample lookup, and engine composition also remain separate.

## Implementation process

The public routing contract was added first and failed on the absent bus roles, route roles,
multi-input view, master preparation, and summing processor. Implementation then extended the
existing edit-versus-prepare split additively, created a routing-specific module to avoid claiming
adjacent mixing controls, and retained exact core channel and sample types plus the concurrency-owned
audio execution domain.

Primary W3C Web Audio material supported explicit channel meaning, summed destination input, and a
single terminal destination. Apple Audio Unit material supported preallocated render resources,
bounded maximum frames, consecutive sample timestamps, and nonblocking real-time render calls.
Superi remains stricter by permitting no implicit channel remap or resample and by precomputing all
route bindings outside the callback.

## Verification

The public `routing_contract` renders two sources through a dry submix and a parallel auxiliary send
and return into the master over adjacent 48 kHz stereo blocks. It proves exact output samples,
stable route inspection, and edit-history-independent floating-point summing order. It also proves
duplicate masters, master outputs, invalid direct, send, and return roles, layout mismatch, and
cycles fail atomically.

The complete audio package suite, existing graph and timeline schedule contracts, warnings-denied
audio Clippy, and generated public documentation pass on Rust 1.97.0 using only
`/private/tmp/superi-targets/P2.W04.C005`. Dependent engine, map, policy, formatting, and full
repository verifier results are retained in the ignored execution record after final synchronization.

## Remaining constraints

The delivered routing path uses unity summing and exact matching channel layouts. Gain, fades, pan,
mute, solo, phase, channel mapping, effects, automation, resampling, metering, and plugin hosting
remain separate checkpoints. No production engine path yet binds scheduled timeline slices and
decoded sample storage into this graph or feeds its master output into the device queue, so this
checkpoint does not claim end-to-end media playback, physical audio behavior, or final export.
